# 核心玩法设计

## 游戏目标

玩家的明面目标：

```text
在金军南下和朝堂崩坏中守住汴京，避免靖康之耻。
```

玩家的隐藏目标：

- 建立一个能听命的朝廷。
- 保住北方关键战线。
- 让勤王军真正抵达。
- 在谈判中掌握主动权。
- 改写关键人物的历史命运。

## 胜负结构

### 失败结局

- **汴京城破**：城防、粮储、战意、城门风险同时崩溃。
- **降议得逞**：主和压力过高，君威过低，朝堂绕过玩家接受苛约。
- **勤王断绝**：北方战线全断，京城孤立，金军威压满值。
- **内乱逼宫**：禁军哗变、宗室恐慌、城中民变共同爆发。
- **财政崩盘**：国库与粮储长期见底，任何守城行动都无法执行。

### 中性结局

- **屈辱和议**：汴京暂保，但割地赔款严重，后续危机埋下。
- **南迁续命**：玩家主动南撤，保住宗室和朝廷，但“朕不南渡”失败。
- **双都割裂**：汴京守住一时，北方大部失守，进入长期对峙。

### 胜利结局

- **汴京保卫成功**：击退或迫退金军，守住京师和宗庙。
- **河东河北未崩**：至少保住若干北方屏障，金军无法直压京城。
- **反攻燕云**：高难路线，要求西军、勤王、财政、外交同时达标。
- **中兴开局**：内政重建，军政改革立住，朝廷摆脱靖康死线。

## 回合尺度

推荐两种尺度：

### 标准模式：一回合一个月

适合长期政略，约 18-30 回合完成一局。

优点：

- 和当前项目兼容。
- 经济、地区、军队变化自然。
- 适合召见和长期记忆。

### 围城模式：一回合十日

当金军抵达汴京城下，临时切成十日一回合。

优点：

- 强化围城压迫感。
- 让城门、粮储、士气、谈判有连续变化。
- 玩家每次行动都像救火。

建议首发：

```text
平时一月一回合，围城时十日一回合。
```

## 每回合流程

```text
1. 朝报
   读本月大事：金军、城防、财政、粮价、派系、地区、密令。

2. 定优先级
   选择本月重点：守城、筹饷、查案、外交、调兵、安民。

3. 召见
   与大臣、将领、密探、地方官、太学生等对话。

4. 密令与调查
   下达暗查、搜证、反间、账册核验、城门排查。

5. 殿前对质
   如果证据足够，可公开审问关键人物。

6. 拟旨
   把玩家意图转为圣旨、军令、榜文、密札。

7. 颁诏
   确认行动并进入执行结算。

8. 月末回奏
   系统展示执行结果、偏差、收益、代价、隐患。

9. 新危机
   触发下一批事件与历史锚点。
```

## 行动预算

为了避免玩家无限对话把所有问题问清楚，建议引入软预算。

每回合默认：

- 召见次数：3 次。
- 朝议次数：1 次。
- 密令新立：1-2 条。
- 殿前对质：最多 1 场。
- 正式旨意：3-5 道。

超出预算可以做，但有成本：

- 君臣疲惫，奏对质量下降。
- 泄密风险上升。
- 朝堂反感，派系亲疏下降。
- 本月执行折扣。

设计意图：玩家不是把所有 NPC 都问一遍，而是在有限时间内判断谁值得信。

## 核心资源

### 顶栏资源

| 资源 | 单位 | 说明 |
| --- | --- | --- |
| 国库 | 万贯 | 朝廷公开财政，军饷、城防、赈济、赏赐从此出 |
| 内帑 | 万贯 | 皇帝可直接动用的私库，适合救急和密支 |
| 京城粮 | 万石 | 汴京粮储，围城时最关键 |
| 民心 | 0-100 | 京城与天下百姓对朝廷的信任 |
| 君威 | 0-100 | 圣旨被执行、朝臣畏服、军队听命的基础 |

### 局势资源

| 资源 | 说明 |
| --- | --- |
| 汴京守备 | 城墙、城门、器械、守军部署综合状态 |
| 金军威压 | 金军距离、兵力、补给、战胜信心 |
| 勤王响应 | 各路军是否愿来、能否来、多久来 |
| 主和压力 | 朝中和议、投降、割地、赔款的政治压力 |
| 城中战意 | 禁军、民壮、士民是否愿守 |

顶栏只放最关键的 5 个，其他放到局势条和详情中，避免 UI 数字过载。

## 事项进度条

事项是玩家长期推动的危机或改革。每条有双向含义：

```text
保卫汴京：-100 = 城防崩溃，+100 = 守备成势
整肃禁军：-100 = 禁军哗变，+100 = 禁军可战
河东防线：-100 = 太原陷落，+100 = 金军受阻
追查军饷：-100 = 账册灭失，+100 = 铁证如山
催发勤王：-100 = 各路观望，+100 = 援军抵近
主和逼宫：-100 = 降议得逞，+100 = 主战路线稳固
```

事项每回合会按惯性变化：

- 没人承办会恶化。
- 低能力承办人会拖慢。
- 高忠诚高能力者会放大进展。
- 缺钱、缺粮、缺兵会让正向旨意打折。
- 触动派系利益会增加反扑事件。

## 玩家行动类型

### 召见

用途：

- 问真相。
- 试探立场。
- 逼人表态。
- 让大臣拟方案。
- 观察其是否推诿。

设计要求：

- 每个大臣都有自己的盲区和私心。
- 回答不应总是可靠。
- 玩家可以拿旧话和证据追问。

### 朝议

多人同场争论。适合：

- 战和大争。
- 守城方案。
- 割地赔款。
- 是否罢免重臣。
- 是否征发民夫。

朝议不是投票，而是暴露派系压力和可执行方案。

### 密令

暗中行动。适合：

- 查账。
- 盯人。
- 搜证。
- 联络边臣。
- 离间金营。
- 排查城门内应。

密令有期限和风险。失败可能泄密。

### 殿前对质

当证据足够，玩家可以把相关人物叫到殿前。

流程：

```text
选择案件 → 选择被质询人 → 选择证据 → 听其辩解 → 追问 → 裁断
```

裁断包括：

- 申饬。
- 罚俸。
- 罢官。
- 下狱。
- 抄没。
- 流放。
- 斩首。
- 留用戴罪。

越重的裁断越爽，但也越容易反噬。

### 拟旨

正式行动。类型包括：

- 任免诏。
- 军令。
- 财政调拨。
- 榜文安民。
- 城防征发。
- 密札。
- 外交国书。

拟旨必须有执行对象、资源来源、期限、承办人。

详见：[丰富诏令类型设计](./11-directive-types.md)。

## 爽点节奏

每个小循环要尽量遵守：

```text
压力出现 → 玩家怀疑 → 获得线索 → 公开或暗中行动 → 即时反馈 → 留下反扑
```

### 示例：禁军欠饷案

压力：

```text
禁军三月欠饷，夜间有士卒聚饮鼓噪。
```

怀疑：

```text
户部称钱已拨，殿前司称未收。
```

线索：

```text
皇城司查到转运司与某宰执门客往来。
```

行动：

```text
殿前对质，出示账册，罢转运使，追银补饷。
```

反馈：

```text
禁军士气 +12，欠饷 -20，君威 +5，主和派亲疏 -8。
```

反扑：

```text
地方转运系统开始拖延粮运，下一月粮价压力上升。
```

## 失败也要好玩

玩家行动失败不能只显示“失败”。要告诉他：

- 哪一步被谁拖了。
- 是钱不够、证据不足、承办人怯懦，还是派系反扑。
- 下次可以如何补救。

失败反馈格式：

```text
旨意原意：补发禁军欠饷。
执行偏差：户部只拨三成，转运司称路费不足。
实际结果：欠饷略降，但士卒仍怨。
暴露线索：有账册显示旧党门客经手此款。
```

这样失败也会变成下一次爽点的线索。

## 终局高潮

终局不能只是一张结局图。应当是连续数回合的大高潮：

1. 金军逼城或和议逼宫。
2. 玩家最后一次朝议。
3. 关键人物根据记忆站队。
4. 玩家颁布最终诏命。
5. 城防、勤王、外交、财政、民心共同结算。
6. 输出史书式结局总结。

结局文案要让玩家觉得：

```text
这是我这一局亲手打出来的历史，不是系统随机给我的判词。
```
