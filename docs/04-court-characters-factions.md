# 朝堂、人物与派系系统

朝堂系统是《朕不南渡》的情绪发动机。玩家面对的不是一组菜单，而是一群会推诿、争功、怕死、结党、藏账、说半句真话的人。

设计目标：

- 让每次召见都像一次政治判断。
- 让每次清算都需要证据和代价。
- 让人物记住玩家的恩、仇、承诺和背叛。
- 让派系成为执行偏差的来源，而不是简单的好坏阵营。

## 人物类型

首发人物建议分为九类。

| 类型 | 玩法作用 |
| --- | --- |
| 宰执 | 国家路线、战和大争、财政军政统筹 |
| 台谏 | 弹劾、舆论、清议、制造或揭露政治压力 |
| 将领 | 守城、勤王、边防、军心 |
| 开封府 | 京城治安、粮价、民壮、案件执行 |
| 皇城司 | 密探、证据、盯人、反间 |
| 户部/转运 | 筹饷、粮运、账册、截留 |
| 工部/军器 | 城防、器械、火油、修城 |
| 太学生/士民代表 | 京城民意、请愿、舆情、战意 |
| 宗室/内廷 | 宫中恐慌、南迁压力、内帑、继承合法性 |

## 核心人物属性

沿用当前项目的四项人物能力，但换成北宋语境。

| 属性 | 说明 |
| --- | --- |
| 忠诚 | 是否愿意为皇帝路线担责，而非口头尊君 |
| 能力 | 是否能把事办成 |
| 清廉 | 是否克扣、卖人情、藏账 |
| 胆略 | 金军压境、群臣反对时是否敢说敢做 |
| 声望 | 新增。影响其建议被朝臣、士民、军队接受的程度 |
| 派系 | 其主要利益网络 |

`声望` 很重要。靖康题材里，有些人物未必最能干，但清议声望高；有些将领能打，却不被文臣信任。声望决定：

- 朝议发言权。
- 担任承办人的政治阻力。
- 被清算时引发的派系反扑。
- 榜文安民和动员勤王的效果。

## 人物状态

建议状态：

| 状态 | 含义 |
| --- | --- |
| active | 在朝或可联络 |
| offstage | 尚未登场或未入玩家视野 |
| summoned | 被召入京或正在路上 |
| dismissed | 罢官 |
| imprisoned | 下狱 |
| exiled | 贬谪/流放 |
| hostage | 为外交或宫廷压力成为质子/人质 |
| surrendered | 投降或附敌 |
| dead | 死亡 |

`hostage` 是靖康题材特有状态。它可以来自：

- 玩家主动以宗室为质拖延。
- 金军谈判索取。
- 城破失败结局。

## 首批人物方向

不要求首发覆盖所有历史人物，但要覆盖足够的戏剧功能。

### 主战核心

- 李纲：守城组织者，高胆略，高声望，容易被主和派攻击。
- 种师道：老将，军事声望高，但年龄、路程和朝廷信任是限制。
- 宗泽：可作为中后期勤王与北方重建核心。

### 主和与自保集团

- 白时中、李邦彦一类主和宰执：不必写成纯反派，可以是“怕亡国、怕担责、相信议和能续命”。
- 城中宗室代表：更关心宗庙与血脉安全，容易推动南迁或和议。

### 旧党与军功旧势力

- 蔡京旧党、童贯旧部：适合做账册、军权、边防责任清算对象。
- 宦官军功集团：掌握旧军中人脉，既有问题，也可能提供短期资源。

### 执行型官员

- 开封府尹：京城治安和粮价。
- 户部尚书：财政调拨。
- 转运使：粮运和军饷。
- 工部官：城防修缮。
- 皇城司使：密探和证据。

### 民间压力人物

- 太学生领袖：清议、请愿、反和舆论。
- 京城商户代表：粮价、借款、囤积。
- 禁军基层代表：欠饷、士气、哗变风险。

## 官职系统

官职不是头衔，而是权限包。人物通过官职获得工具。

| 官职类型 | 可用能力 |
| --- | --- |
| 宰执 | 发起朝议、统筹多部门、推荐任免 |
| 台谏 | 弹劾、调查风闻、制造清议压力 |
| 开封府 | 抓捕、维稳、查粮价、组织民壮 |
| 皇城司 | 密查、盯梢、搜证、反间 |
| 户部 | 拨款、借款、查账、追缴 |
| 转运 | 调粮、押运、路费核算 |
| 殿前司 | 禁军部署、城门守备、军纪 |
| 工部 | 修城、军器、火油、壕沟 |
| 宣抚/制置 | 地方军政、勤王路线、边防协调 |

皇帝可以临时授权，但会有后果：

- 给李纲过大军政权，会提高守城效率，也会招来“专权”攻击。
- 给皇城司过大权，会提高查案效率，也会降低士大夫亲疏。
- 给开封府强力抓捕权，会压住粮价，也可能激起商户反弹。

## 派系系统

派系不是静态标签，而是利益网络。

本章只说明朝堂中的派系入口；完整数值、反扑时钟、执行链路和 UI 设计见：[强派系系统设计](./10-advanced-faction-system.md)。

### 首批派系

| 派系 | 诉求 | 玩法作用 |
| --- | --- | --- |
| 主战清议 | 守城、拒割地、重用忠臣 | 提供民意和君威，但可能推动冒险 |
| 主和宰执 | 保宗庙、避决战、议和 | 能拖延金军，也可能滑向屈辱 |
| 旧党门生 | 保既得利益、遮掩旧账 | 截留、拖延、反扑清算 |
| 宦官军功集团 | 保军权、保旧功 | 掌握禁军旧关系，风险高 |
| 西军边臣 | 要钱粮、要信任、要军权 | 真正能打，但调动困难 |
| 京城士民 | 要粮价稳定、要守城信心 | 决定战意和民变 |
| 宗室内廷 | 要安全、要体面、怕城破 | 推动南迁、质子、和议 |
| 转运财税网络 | 要保账、保路线、保油水 | 决定钱粮是否到位 |

### 派系数值

| 数值 | 说明 |
| --- | --- |
| 影响力 | 能影响多少人物、舆论、地方、军队 |
| 亲疏 | 对玩家路线的支持 |
| 利益受损 | 最近被政策伤到多少 |
| 反弹风险 | 弹劾、拖延、泄密、串联的概率 |
| 恐惧 | 被清算震慑的程度 |

`恐惧` 是爽点服务数值。清算成功会提高恐惧，让短期执行顺畅；恐惧过高则可能导致抱团反扑、毁证、逃亡。

## 召见玩法

### 召见目标

玩家召见不是为了刷好感，而是为了判断：

- 他知道什么。
- 他想隐瞒什么。
- 他害怕谁。
- 他愿意为哪条路线担责。
- 他是否适合承办。

### 对话工具

人物 agent 可调用的工具应与官职绑定：

- `propose_directive`：提出草案。
- `request_investigation`：建议密查。
- `offer_testimony`：提供证词。
- `recommend_appointment`：举荐人物。
- `challenge_claim`：质疑另一人的说法。
- `recall_memory`：回忆旧事。

### 追问机制

玩家可以拿三类内容追问：

- 证据：账册、密奏、证词。
- 旧话：上月某人承诺过什么。
- 结果：某道旨意为何执行偏了。

追问成功会产生：

- 新线索。
- 人物态度变化。
- 对质案件条件。
- 派系反应。

## 朝议玩法

朝议用于大问题，不是日常闲聊。

适合触发朝议的议题：

- 是否坚守汴京。
- 是否割让三镇。
- 是否罢免某重臣。
- 是否把禁军交给主战将领。
- 是否征发民夫修城。
- 是否开放内帑补军。

朝议结构：

```text
议题 → 各派发言 → 玩家插问 → 关键人物表态 → 可执行方案列表
```

输出不应是“最佳答案”，而是：

- 方案 A 的收益和政治代价。
- 方案 B 的收益和军事风险。
- 谁支持，谁反对，谁沉默。
- 哪些信息仍不可靠。

## 殿前对质系统

这是本游戏最重要的爽点系统。

### 触发条件

至少满足一个：

- 密令查得证据。
- 两名以上人物证词冲突。
- 账本和实际结果明显矛盾。
- 事项失败且承办人解释不清。
- 历史锚点触发重大问责。

### 案件结构

| 字段 | 说明 |
| --- | --- |
| case_id | 案件 id |
| title | 案名，如“禁军欠饷截留案” |
| suspects | 涉案人物 |
| evidence | 证据列表 |
| stakes | 牵涉利益 |
| public_pressure | 公开审理压力 |
| risk | 反噬风险 |

### 证据等级

| 等级 | 说明 |
| --- | --- |
| 风闻 | 可追问，不足以定罪 |
| 证词 | 可施压，可能被反驳 |
| 账册 | 强证据，但可能伪造或残缺 |
| 手书 | 强证据，牵涉人物更广 |
| 人赃俱获 | 可重判 |
| 多源闭环 | 最适合公开清算 |

### 对质过程

```text
1. 宣读案名
2. 被质询人初辩
3. 玩家出示证据
4. 相关人物插话或拆台
5. 被质询人认罪、狡辩、反咬或沉默
6. 玩家裁断
7. 即时反馈与派系反应
```

### 裁断后果

轻判：

- 派系反弹低。
- 君威收益低。
- 当事人可能继续埋雷。

重判：

- 爽点强。
- 君威和恐惧上升。
- 相关派系利益受损。
- 可能导致毁证、串联、怠工。

### 留用戴罪

这是很适合北宋题材的选择。玩家可以抓住把柄，但不立刻杀人。

效果：

- 当事人短期忠诚上升或恐惧上升。
- 执行力可能提升。
- 清议可能不满。
- 若玩家失势，此人极可能反咬。

## 人物成长与命运改写

玩家可以改变人物命运：

- 提前重用。
- 免于被贬。
- 戴罪立功。
- 从小官提拔为战线核心。
- 被冤杀或被误信。

每个核心人物应有“命运节点”：

```text
李纲是否被罢？
种师道能否及时入援？
宗泽是否获得足够授权？
某主和宰执是否被清算或转向？
某禁军将领是否哗变？
```

命运节点会写入结局史书，增强玩家“这是我的历史”的感觉。
