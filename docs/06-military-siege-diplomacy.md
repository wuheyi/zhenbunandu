# 军事、围城与外交系统

《朕不南渡》的军事玩法不追求复杂战棋，而是让玩家在政略层面决定“谁来打、有没有钱粮、守哪里、何时谈、何时硬”。战斗细节通过战报和关键节点呈现，核心是压力、准备和执行偏差。

## 军事系统目标

- 让金军南下形成持续压迫。
- 让汴京城防成为可经营盘面。
- 让勤王军不是自动救场，而是需要玩家催动。
- 让战和谈判互相影响。
- 让玩家的准备在战报中得到明确回报。

## 军队类型

| 类型 | 说明 |
| --- | --- |
| 禁军 | 京城主力，人数多但训练、军纪、欠饷问题严重 |
| 西军 | 北宋最可靠的野战力量，调动慢、要钱粮和信任 |
| 河东军 | 太原等地防线，牵制金军 |
| 河北军 | 金军南下首当其冲，容易溃散或被截断 |
| 勤王军 | 各地响应而来，质量不一 |
| 民壮 | 京城士民组织，守城有用，野战弱 |
| 金军东路/西路 | 外部压力主体 |
| 金营诸部 | 可被离间、收买、挑拨争功 |

## 汴京围城盘面

这是新游戏的核心差异系统。

### 总览指标

| 指标 | 范围 | 说明 |
| --- | --- | --- |
| 城防 | 0-100 | 城墙、城门、壕沟、器械综合 |
| 守军战意 | 0-100 | 禁军、民壮、士民守城意愿 |
| 京城粮储 | 万石 | 能撑多久 |
| 城门风险 | 0-100 | 内应、怯守、夜开门、混乱 |
| 火患 | 0-100 | 攻城火、城中纵火、仓储火灾 |
| 降议压力 | 0-100 | 城中要求议和投降的压力 |
| 金军威压 | 0-100 | 敌军兵力、距离、攻势、士气 |

### 城段与城门

可把汴京抽象成若干关键节点：

- 宣化门。
- 陈州门。
- 新宋门。
- 封丘门。
- 汴河水门。
- 皇城外垣。
- 粮仓区。

每个节点字段：

| 字段 | 说明 |
| --- | --- |
| condition | 坚固度 |
| garrison | 驻守兵力 |
| commander | 守将 |
| risk | 内应/失守风险 |
| equipment | 弓弩、炮石、火油 |

玩家不需要微操每个兵种，但可以做关键部署：

- 加固某门。
- 派可信将领驻守。
- 给某城段拨器械。
- 排查内应。
- 设置预备队。
- 组织民壮搬运。

## 城防行动

| 行动 | 消耗 | 收益 | 风险 |
| --- | --- | --- | --- |
| 修城墙 | 钱、民夫、时间 | 城防上升 | 民怨、工期拖延 |
| 储炮石火油 | 军资 | 攻城战收益 | 火患上升 |
| 编练民壮 | 粮、声望 | 战意与守备上升 | 训练不足可能混乱 |
| 排查城门 | 皇城司/开封府 | 城门风险下降 | 误抓伤民心 |
| 夜禁戒严 | 君威 | 内应风险下降 | 民心下降 |
| 开仓劳军 | 粮 | 士气上升 | 粮储下降 |
| 重赏敢死 | 钱 | 短期战意上升 | 国库下降 |

## 战役与战报

战斗不是实时操控，而是事件化结算。

### 战役触发

- 金军抵达某地区。
- 玩家命令出战。
- 守城节点被攻击。
- 勤王军遭遇截击。
- 玩家派兵袭营或断粮。

### 战役输入

- 双方军队数值。
- 主将属性。
- 地区地形。
- 粮草补给。
- 天气和时令。
- 玩家前期准备。
- 相关记忆和士气。

### 战报输出

战报要分层：

```text
结论：胜 / 小胜 / 相持 / 小败 / 大败
原因：哪些准备起作用，哪些问题拖后腿
损失：兵力、士气、器械、粮草
后续：敌军威压、勤王响应、谈判条件变化
```

示例：

```text
金军夜攻宣化门，守将按前诏预置火油与炮石。
初更敌骑逼城，民壮一度惊散，李纲亲临城上，禁军弩手齐发。
三鼓后敌军退。城墙小损，守军伤亡七百，金军攻势受挫。
```

结构化变化：

- 城防 -3。
- 守军战意 +8。
- 金军威压 -6。
- 粮储 -2 万石。
- 李纲声望 +5。

## 勤王系统

勤王军是“希望”，但不是自动按钮。

### 勤王响应流程

```text
诏令发出 → 地方是否相信朝廷能守 → 将领是否愿担责 → 路线是否安全 → 钱粮是否足 → 是否按期抵达
```

### 勤王字段

| 字段 | 说明 |
| --- | --- |
| origin | 起兵地区 |
| commander | 将领 |
| manpower | 兵力 |
| readiness | 准备度 |
| loyalty | 听命程度 |
| route | 路线 |
| eta | 预计到达回合 |
| obstruction | 阻碍 |
| status | 观望/集结/行军/受阻/抵达/溃散 |

### 玩家行动

- 发勤王诏。
- 给赏格。
- 点名主将。
- 派使者催促。
- 改路线。
- 调粮草接应。
- 给地方承诺。
- 公开战报增强信心。

### 反向事件

- 地方称道路被金军截断。
- 主将畏罪观望。
- 粮草不继。
- 主和派暗示“不必轻动”。
- 金军分兵阻援。

## 河东与太原

河东防线是首发重要战线。它的作用不是让玩家在地图上全面作战，而是形成战略牵制：

- 河东守住，金军威压下降。
- 河东失守，汴京压力加速上升。
- 玩家救河东需要钱粮、将领、勤王和外交牵制。

事项示例：

```text
河东屏障
  -100 太原陷落，金军西路南压
   0   围困相持
  +100 河东稳住，金军被迫分兵
```

## 金宋外交

外交不是和平按钮，而是军事系统的一部分。

### 金方诉求

金方可能提出：

- 金银赔款。
- 割让三镇。
- 交出质子。
- 罢免主战大臣。
- 开城犒军。
- 承认宗藩关系。

### 外交字段

| 字段 | 说明 |
| --- | --- |
| demand_severity | 条件苛刻程度 |
| trust | 金方是否相信宋廷承诺 |
| impatience | 金方是否准备攻城 |
| internal_tension | 金营内部争功或分歧 |
| leverage | 宋方谈判筹码 |

### 玩家策略

| 策略 | 收益 | 风险 |
| --- | --- | --- |
| 真议和 | 买时间 | 条件沉重，君威下降 |
| 假议和 | 拖延勤王 | 识破后金军更强硬 |
| 强硬拒绝 | 君威战意上升 | 金军攻势上升 |
| 分化金营 | 降低攻势 | 需要情报密支 |
| 交部分财物 | 暂缓攻城 | 财政崩盘 |
| 斩使或辱使 | 爽点强 | 高风险战争升级 |

## 战和互动

战场胜利会改善谈判条件：

- 金军威压下降。
- 宋方 leverage 上升。
- 主和压力下降。
- 勤王响应上升。

议和拖延会改善战场准备：

- 多一回合修城。
- 多一回合调粮。
- 多一回合催勤王。

但拖延过度会反噬：

- 君威下降。
- 主战派不满。
- 金方识破。
- 城中流言。

## 军事爽点

军事爽点应来自“准备兑现”。

例：

```text
上月玩家力排众议，把火油拨给宣化门。
本月金军夜攻，火油发挥作用。
战报明确写出：若无前诏，宣化门几乎失守。
```

玩家会觉得：

```text
这不是随机赢，是我提前布置赢的。
```

## 首发战役事件

建议首批做 8 类：

1. 金军逼近黄河。
2. 河北边军溃报。
3. 河东求援。
4. 汴京第一次戒严。
5. 金使入城索约。
6. 夜攻城门。
7. 勤王军受阻。
8. 城中降议逼宫。

这 8 类足够组成完整靖康危机。
