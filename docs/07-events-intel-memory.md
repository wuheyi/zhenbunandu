# 事件、情报、密令与记忆系统

本系统负责把历史危机送到玩家面前，并让玩家的调查和决策留下长期痕迹。

参考当前项目，事件负责“抛问题”，密令负责“查真相”，记忆负责“让世界记得”，历史锚点负责“保持靖康压力”。

## 事件系统

事件不是随机新闻，而是可行动的问题。

每个事件应包含：

| 字段 | 说明 |
| --- | --- |
| id | 事件 id |
| title | 标题 |
| kind | 军事/财政/朝政/地方/外交/城防/人物 |
| summary | 玩家可见摘要 |
| urgency | 是否必须本月处理 |
| severity | 不处理的伤害 |
| credibility | 奏报可信度 |
| interests | 牵涉派系和人物 |
| clues | 可调查线索 |
| actions | 推荐可执行方向 |
| trigger | 年月、数值阈值或历史锚点 |
| hidden_truth | 可选隐藏真相 |

## 事件类型

### 军事事件

- 金军南下。
- 边城失守。
- 禁军鼓噪。
- 勤王受阻。
- 城门守将请援。

### 财政事件

- 军饷不明。
- 粮船未至。
- 商户囤积。
- 内帑争议。
- 赔款索求。

### 朝政事件

- 主和派弹劾主战大臣。
- 台谏联名请斩误国者。
- 宗室请求南幸。
- 宰执互相推责。

### 城防事件

- 城门内应传闻。
- 火油库失火。
- 民夫逃散。
- 修城料不足。
- 夜禁冲突。

### 外交事件

- 金使索地。
- 金营内部争功。
- 西夏观望。
- 辽残余求联络。

### 人物事件

- 某将请缨。
- 某臣请辞。
- 某人密奏。
- 某人被诬。
- 某人家属外逃。

## 历史锚点

历史锚点用于保证靖康压力，不用于强行锁死结局。

锚点示例：

| 阶段 | 锚点 | 作用 |
| --- | --- | --- |
| 开局 | 金兵南下消息 | 建立倒计时 |
| 前期 | 第一次汴京危机 | 检验城防和外交 |
| 中期 | 河东/太原压力 | 牵动西路金军 |
| 中期 | 主和派逼宫 | 检验朝堂控制 |
| 后期 | 金军再临 | 终局压力 |
| 终局 | 汴京攻防/和降抉择 | 导向结局 |

每个锚点要有条件分支：

```text
如果河东防线高：金军西路受阻，汴京压力减弱。
如果城防高：金军偏向谈判或围困。
如果主和压力高：锚点转为逼宫事件。
如果勤王响应高：锚点转为野战或解围机会。
```

## 密令系统

密令是玩家把怀疑变成证据的主要方式。

### 密令类型

| 类型 | 用途 |
| --- | --- |
| 查账 | 军饷、粮运、赔款、内帑 |
| 盯人 | 观察大臣、将领、商户、内廷 |
| 搜证 | 找账册、手书、证词 |
| 反间 | 金营、主和派、敌方使者 |
| 排查 | 城门、粮仓、军营 |
| 联络 | 勤王将领、地方官、辽残余 |
| 安插 | 在敌营、商帮、转运网络布线 |

### 密令字段

| 字段 | 说明 |
| --- | --- |
| title | 密令标题 |
| assignee | 承办人 |
| content | 任务内容 |
| tags | 人物、地区、事项关键词 |
| due_turn | 到期回合 |
| secrecy | 保密程度 |
| risk | 泄密风险 |
| progress | 进度 |
| status | active/done/failed/leaked |
| result | 回奏 |

### 承办人选择

不同承办人有不同风格：

- 皇城司：查人强，容易伤士大夫亲疏。
- 开封府：京城执行强，但出京弱。
- 台谏：舆论强，证据可能不扎实。
- 转运官：账册强，但可能自保。
- 将领亲兵：军中强，朝堂弱。
- 商人线人：粮价和金钱强，忠诚低。

## 证据系统

证据是殿前对质的燃料。

### 证据来源

- 密奏。
- 账册。
- 证词。
- 手书。
- 金营情报。
- 城门值夜记录。
- 粮仓出入簿。
- 旧旨执行回执。

### 证据字段

| 字段 | 说明 |
| --- | --- |
| evidence_id | id |
| title | 名称 |
| kind | 账册/证词/手书/物证/情报 |
| strength | 证据强度 |
| reliability | 可信度 |
| source | 来源 |
| implicated | 涉及人物/派系 |
| usable_in_court | 是否可公开 |
| risk_if_revealed | 公开风险 |

### 公开与秘密

有些证据强但不能公开：

- 暗线身份会暴露。
- 牵涉宗室。
- 牵涉外交承诺。
- 证据来自伪造或诱供。

玩家可以选择：

- 公开清算，爽点强。
- 私下胁迫，收益稳。
- 继续深挖，风险上升。

## 记忆系统

记忆让世界有连续性。

### 记忆主体

- 人物。
- 派系。
- 地区。
- 军队。
- 城门/城段。
- 外部势力。
- 朝廷整体。

### 记忆类型

| 类型 | 示例 |
| --- | --- |
| appointment | 玩家任命李纲掌城防 |
| punishment | 某转运使被抄没 |
| promise | 玩家许诺补发西军军饷 |
| betrayal | 玩家议和后又撕约 |
| battle | 宣化门夜战获胜 |
| investigation | 皇城司查得账册 |
| mercy | 某臣戴罪留用 |
| failure | 勤王诏无人响应 |

### 记忆字段

| 字段 | 说明 |
| --- | --- |
| subject_type | character/faction/region/army/power/court |
| subject_id | 主体 |
| turn | 回合 |
| title | 摘要 |
| cause | 起因 |
| process | 经过 |
| outcome | 结果 |
| sentiment | 正/负/混合 |
| importance | 1-5 |
| tags | 检索关键词 |
| expires_turn | 过期回合，重要记忆永久 |

## 记忆影响玩法

### 召见

人物会提起旧事：

```text
“陛下上月许臣三万贯以募民壮，今户部只拨一万。臣若再催百姓，恐难服众。”
```

### 朝议

派系会引用旧账：

```text
主和派：上月强硬拒使，金军果然攻城，臣等岂是无故言和？
```

### 执行

承办人记得玩家是否兑现承诺：

- 兑现承诺 → 忠诚和执行上升。
- 反复失信 → 阳奉阴违。
- 冤枉忠臣 → 主战派亲疏下降。
- 戴罪留用 → 当事人恐惧上升。

### 结局

结局史书引用关键记忆：

```text
史臣曰：帝初立，力排群议，用李纲守京师；又发内帑补禁军，斩克饷者于殿前。故金人再围，城中犹能死守。
```

## 情报与误报

不是所有奏报都是真的。

可信度来源：

- 来源人物能力与清廉。
- 是否多源印证。
- 是否符合当前盘面。
- 是否牵涉其自身利益。
- 是否来自敌方反间。

玩家可用密令验证。

误报不能只是坑玩家。它应该成为玩法：

- 抓到造谣者。
- 发现敌方反间。
- 暴露派系操作。
- 迫使玩家建立情报网络。

## 事件生成规则

每回合事件来源：

1. 历史锚点。
2. 数值阈值。
3. 事项进度。
4. 密令结果。
5. 人物记忆。
6. 随机扰动。

优先级：

```text
历史锚点 > 失败阈值 > 玩家行动反扑 > 事项推进 > 随机事件
```

每月给玩家 3-5 条关键奏报即可。不要把所有后台变化都塞给玩家。

## 爽点事件模板

### 查到线索

```text
皇城司夜报：禁军欠饷案已有眉目。
某转运判官家仆供称，军饷曾转入相府门客名下。
但账册原本藏在汴河东仓，若公开搜检，恐惊动其党。
```

### 殿前对质

```text
户部称饷已发，殿前司称未收。
陛下命取东仓账册，当殿核对。
某臣初辩“路费损耗”，继而被家仆证词击破。
```

### 旧恩回报

```text
上月陛下保下某将未被主和派弹劾。
今金军夜攻，其率所部死守城门，战至天明。
```

### 失信反噬

```text
陛下曾许西军入援后补饷，至今未拨。
种师道部行军迟缓，军中已有怨言。
```
