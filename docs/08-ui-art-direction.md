# UI 风格与细节设计

《朕不南渡》的 UI 目标是：一打开就进入“靖康前夜的政务桌面”，而不是看到一个宣传页。玩家第一眼应当看见汴京、金军压力、朝报、朝臣和待办旨意。

关键词：

```text
北宋院体、绢本舆图、御案奏牍、朱印裁断、汴京围城、清明上河图式横向信息流
```

## 总体风格

### 不是泛中国风

不要做成通用水墨武侠，也不要照搬明清宫廷。北宋气质应更接近：

- 绢本底色。
- 院体山水和界画线条。
- 清雅、克制、文官政务感。
- 朱印、墨批、青绿山水点色。
- 汴京市井与宫廷并置。

### 视觉参考方向

- 《清明上河图》的横向城市叙事感。
- 宋代院体画的细密线条与留白。
- 古籍版式里的朱批和栏线。
- 舆图、城防图、账册、奏札、军令牌。

### 调色板

| 用途 | 颜色 | 说明 |
| --- | --- | --- |
| 绢纸底 | `#F1E6CD` | 主背景，温润但不偏黄过度 |
| 旧纸阴影 | `#D8C49A` | 分区、卷轴边缘 |
| 墨色 | `#1F2522` | 正文 |
| 淡墨 | `#5F655D` | 次级文本 |
| 朱批 | `#A7342B` | 危机、盖印、确认 |
| 金军警色 | `#7D2E24` | 敌军压力，不用荧光红 |
| 青绿山水 | `#2F6F73` | 地图水系、可行动状态 |
| 宋瓷青 | `#9CB9B0` | 次级高亮 |
| 乌金 | `#3A3026` | 顶栏和深色按钮 |
| 米白 | `#FFF8E8` | 弹层和奏报主体 |

避免：

- 大面积纯黑。
- 霓虹红。
- 高饱和金色。
- 现代玻璃拟态。
- 大面积紫蓝渐变。

## 字体

建议：

- 标题：宋体/仿宋风格，优先 `Noto Serif SC`。
- 正文：`Noto Serif SC` 或系统宋体。
- 数字：可用清晰现代数字字体，但颜色和尺寸融入整体。
- 关键盖印：可用篆刻风图片或专门字体，但不承担主要阅读。

原则：

- 不做大段竖排正文，影响可读性。
- 奏折标题可短竖排或印章化。
- 按钮文字要清晰，不追求古风到难读。

## 主界面布局

首屏即游戏主界面。

本章定义视觉风格与主布局；更细的地图交互、奏疏阅读、人物关系和月末报告流程见：[前端体验设计](./14-frontend-experience.md)。

```text
┌──────────────────────────────────────────────┐
│ 顶栏：年月 / 国库 / 内帑 / 京城粮 / 民心 / 君威 │
├──────────────┬───────────────────┬───────────┤
│ 左：朝报奏札  │ 中：汴京与北方舆图  │ 右：朝臣席 │
│ 危机倒计时    │ 城防/军线/粮道       │ 派系/召见 │
│ 事项进度      │                   │           │
├──────────────┴───────────────────┴───────────┤
│ 底部：御案操作区：召见 / 密令 / 对质 / 拟旨 / 颁诏 │
└──────────────────────────────────────────────┘
```

### 顶栏

顶栏像御案上方的窄条木匾，不要做现代 dashboard。

显示：

- 当前年月：靖康元年正月。
- 阶段：朝报 / 召见 / 拟旨 / 结算 / 围城。
- 国库：`42 万贯`。
- 内帑：`18 万贯`。
- 京城粮：`31 万石`。
- 民心：`62`。
- 君威：`48`。

细节：

- 资源变化用小号朱/青标记：`+8`、`-12`。
- 鼠标悬停显示上月流水摘要。
- 资源低于阈值时不闪烁，用朱批小印“急”标记。
- 顶栏高度稳定，不因数字变化抖动。

### 左栏：朝报与危机

左栏像叠放的奏札。

模块：

1. 亡国倒计时。
2. 本月急奏。
3. 进行中事项。
4. 密令到期。

#### 亡国倒计时

不要只显示日期，要显示压力路径：

```text
金军南下：逼近黄河
汴京危机：约 2 回合
河东屏障：危
主和压力：升
```

用横向短条表示：

- 金军威压。
- 勤王响应。
- 主和压力。
- 汴京守备。

#### 朝报条目

每条朝报像一张小奏札：

```text
边报
金军前锋抵近黄河
可信：中  牵涉：河北 / 河东 / 金军
```

交互：

- 点击后在中区打开详情。
- 可直接“召见相关人”。
- 可直接“立密令”。
- 可加入“本月重点”。

不要把朝报做成过多现代卡片阴影。用纸面、细线、朱批角标即可。

### 中区：汴京与北方舆图

中区是主视觉。

默认视图为“北方局势图”：

- 汴京在下中偏右。
- 黄河、太原、真定、中山、河东、河北、陕西路以舆图形式出现。
- 金军路线用暗朱箭线。
- 勤王路线用青绿色线。
- 粮道用淡蓝/米色虚线。
- 危险节点带朱印小标。

切换视图：

- 天下。
- 汴京。
- 城防。
- 粮道。
- 勤王。
- 外交。

#### 汴京视图

汴京视图更像城防平面图。

显示：

- 城墙轮廓。
- 城门节点。
- 粮仓区。
- 皇城。
- 汴河水门。
- 金军营地位置。
- 民壮集结点。

城门节点状态：

- 稳：青瓷色。
- 危：朱批边。
- 疑：墨色问号。
- 破：深朱裂纹。

点击城门打开详情：

```text
宣化门
城段坚固：67
驻守：殿前司三千 / 民壮八百
守将：李纲临督
器械：炮石足，火油少
风险：夜袭
可行动：加派守军 / 拨火油 / 排查内应
```

### 右栏：朝臣席

右栏像殿上班列，不是普通联系人列表。

分组：

- 宰执。
- 台谏。
- 军方。
- 财赋。
- 京城。
- 密探。
- 宗室内廷。

每个人物条显示：

- 姓名。
- 官职。
- 立场短标：主战、主和、自保、财赋、边臣。
- 忠诚/能力的模糊评价，不直接全数值裸露。
- 本月状态：可召见、已召见、在外、下狱、称病。

例：

```text
李纲
兵部侍郎 · 主战
敢任事 / 受攻讦
```

悬停显示：

- 近期记忆。
- 支持/反对的事项。
- 可用工具。

点击进入召见。

### 底部：御案操作区

底部是玩家的“手”。

固定按钮：

- 召见。
- 朝议。
- 密令。
- 对质。
- 拟旨。
- 颁诏。
- 史册。
- 存档。

按钮风格：

- 木牌或纸签，不做大圆角胶囊。
- 激活状态用朱批下划线。
- 危险操作如“颁诏”用朱印按钮。
- 图标可用简洁线性图标，但要贴合：奏札、印、令箭、眼、城门。

## 关键界面

### 召见界面

布局：

```text
左：人物立绘与档案
中：对话正文
右：相关记忆 / 可问事项 / 可用证据
底：输入框与快捷意图
```

细节：

- 人物立绘建议半身像，宋代官服，不要现代写实照片感。
- 对话背景像展开的素纸，不用聊天气泡过重。
- 玩家输入框像御批栏。
- 快捷意图用短签：
  - 追问账册
  - 要他表态
  - 命其拟案
  - 拿证据质问
  - 许诺
  - 威胁

人物回答要保留普通横排文本，保证长文可读。

### 朝议界面

朝议是多人同屏。

视觉：

- 中间议题牌。
- 左右两列或半环形显示人物。
- 发言时人物牌亮起。
- 派系立场用细色边，不用夸张阵营色。

功能：

- 玩家可点名追问。
- 可把一句发言钉为“证词”。
- 可把冲突转成“对质案件”。
- 朝议结束输出方案列表。

### 殿前对质界面

这是最需要爽感的界面。

布局：

```text
上：案名与牵涉利益
左：被质询人
中：对质过程
右：证据架
底：裁断区
```

视觉细节：

- 证据像一份份账册/密札，可拖入中间出示。
- 出示强证据时盖半透明朱印“验”。
- 被质询人辩解后，可出现“破绽”标记。
- 裁断按钮像御前朱批：
  - 申饬
  - 罢官
  - 下狱
  - 抄没
  - 斩
  - 戴罪留用

裁断后不要只弹 toast。要有短暂的“御印落下”动效，然后显示结果：

```text
裁断已行
国库 +18 万贯
禁军欠饷 -21
君威 +6
旧党恐惧 +14
转运阻力 +9
```

### 拟旨界面

拟旨界面像御案上的诏书草案。

结构：

- 左：草案列表。
- 中：正式文本。
- 右：结构化效果预览。

每道旨意必须展示：

- 目标。
- 承办人。
- 资源来源。
- 期限。
- 风险。

例：

```text
旨意：补发禁军欠饷
承办：户部尚书 + 开封府
资源：国库 20 万贯，内帑 5 万贯
期限：本月
风险：转运网络抵触
```

### 月末回奏界面

月末回奏是玩家获得反馈的高潮。

结构：

1. 本月大势一句话。
2. 每道旨意的执行结果。
3. 财政流水摘要。
4. 军事/城防变化。
5. 派系和人物记忆。
6. 下月危机预告。

每道旨意结果格式：

```text
原旨：命李纲总督城防，三日内修宣化门。
执行：李纲亲至城上，工部拨料不足，开封府强征民夫。
结果：宣化门坚固上升，民心小降，主和派弹劾“扰民”。
```

### 史册界面

史册不是普通日志。它应该像玩家这一局的历史书。

分类：

- 大事年表。
- 诏令。
- 对质案件。
- 战报。
- 人物命运。
- 结局史论。

玩家可以回看：

- 某人为何恨我。
- 某道旨意为何失败。
- 某个证据来自哪里。

## 信息层级

界面中不要同时展示所有数值。

### 第一层：必须一直看见

- 国库。
- 内帑。
- 京城粮。
- 民心。
- 君威。
- 金军威压。
- 汴京守备。
- 主和压力。

### 第二层：点击才看

- 各军欠饷、士气、训练。
- 各地区税收、粮储、路线风险。
- 各派系亲疏、恐惧、反弹。
- 各城门状态。

### 第三层：日志里看

- 每笔流水。
- 每次属性 delta。
- LLM 详细推演痕迹。
- 证据来源摘录。

## 状态表达

### 危机

用朱批、裂纹、短促文案。

```text
急
危
疑
断
```

### 成功

用青绿色和淡金，不用游戏化大绿勾。

```text
已达
稍安
可战
```

### 不确定

用淡墨问号和“待核”。

```text
风闻
待核
有疑
```

## 动效

动效要少但有仪式感。

推荐：

- 朱印落下：确认裁断/颁诏。
- 奏札展开：打开朝报详情。
- 舆图箭线推进：金军或勤王行军。
- 墨迹晕开：新危机出现。
- 账册翻页：财政流水。

避免：

- 频繁闪烁。
- 大量粒子。
- 现代弹窗飞入。
- 过度拟物导致操作慢。

## 响应式

桌面优先，但要保证小屏可用。

### 桌面

- 三栏布局。
- 中区地图最大。
- 右栏人物常驻。

### 窄屏

- 顶栏横向滚动或折行。
- 主区以标签切换：朝报 / 舆图 / 朝臣 / 御案。
- 底部操作区固定。
- 对质和拟旨用全屏面板。

## 可访问性

- 所有颜色状态必须配文字，不只靠颜色。
- 古风字体只用于标题和短标签，正文保证清晰。
- 关键数字至少 14px，正文至少 15-16px。
- 不使用负字距。
- 所有长按钮允许换行，不挤压文字。
- 证据强弱用文字和图形双重表达。

## 美术资产需求

首发最少需要：

- 一张北宋北方舆图底图。
- 一张汴京城防抽象图。
- 20-30 张人物半身立绘。
- 奏札/账册/诏书/朱印 UI 纹理。
- 城门、军营、粮船、金军、勤王路线图标。
- 结局背景 4-6 张。

如果预算有限，优先顺序：

1. 汴京/北方舆图。
2. 人物立绘。
3. 奏札和朱印组件。
4. 结局图。

详细素材清单和逐项画图提示词见：[美术素材提示词](./15-art-asset-prompts.md)。

## 首屏体验

新局开始后的第一屏应当是：

```text
顶栏显示：靖康元年正月，国库、内帑、京城粮、民心、君威。
中间舆图上金军箭线逼近黄河。
左栏第一条急奏：金军南下。
右栏李纲、主和宰执、户部、皇城司可召见。
底部“召见”“密令”“拟旨”可用，“颁诏”暂灰。
```

玩家不用读教程，就知道：

```text
敌人快来了，我得先问人、查账、下旨。
```

## UI 质量标准

一个界面算合格，必须满足：

- 玩家 3 秒内知道当前最大危机。
- 玩家 10 秒内能找到下一步可操作按钮。
- 数值变化能追溯到具体旨意或事件。
- 任何古风装饰都不妨碍阅读。
- 关键爽点有专门界面承接，而不是普通文本飘过。
