# MVP 路线与内容清单

本文定义《朕不南渡》的首发范围。目标不是一开始做完整北宋模拟器，而是先做出能让玩家上头的核心体验：

```text
靖康高压短局 + 朝堂清算 + 汴京围城 + 金宋谈判
```

## MVP 成功标准

玩家完成第一局后，应能说出至少一个这样的故事：

- 我查出禁军军饷被截留，当朝抄了相关官员。
- 我力排众议让李纲守城，金军夜攻被打退。
- 我用假议和拖到勤王军抵近，反手拒绝割地。
- 我错信主和派，导致城门风险暴涨，最后被逼签了屈辱和约。
- 我内帑补军稳住了禁军，但宗室和内廷开始逼我南迁。

如果玩家只记得“点了几个按钮，数值涨跌”，MVP 就失败了。

## 首发剧本范围

### 时间范围

默认：

```text
靖康元年正月 → 靖康二年春
```

约 15-20 个月。围城阶段可拆成十日回合。

### 地图范围

首发只做关键区域：

- 汴京。
- 京畿。
- 河北东路。
- 河北西路。
- 河东路。
- 陕西路。
- 淮南。
- 江南/两浙。

### 战线范围

首发聚焦：

- 汴京守城。
- 河东屏障。
- 勤王入援。
- 金军东/西两路压力。

不做完整全国战争。

## MVP 子系统优先级

### P0 必须有

- 新局/存档/LLM 配置。
- 回合推进。
- 朝报。
- 人物召见。
- 草案/拟旨/颁诏。
- 月末流式结算。
- 国库、内帑、京城粮、民心、君威。
- 汴京守备、金军威压、主和压力、勤王响应。
- 事项进度条。
- 密令。
- 证据。
- 殿前对质。
- 城防战报。
- 结局判定。

### P1 强烈建议

- 朝议多人辩论。
- 人物记忆。
- 财政流水。
- 粮价指数。
- 城门节点。
- 金宋谈判。
- 勤王路线。
- 史册回放。
- 自动跑局与平衡报告。
- 首局新手引导和失败复盘。

### P2 后续扩展

- 多剧本。
- 更多宋代制度细节。
- 南迁路线。
- 北伐路线。
- 更复杂外交。
- 人物成长树。
- 皇帝技能树。
- Steam 成就。

## 首批人物清单

MVP 约 30 人足够。

### 朝堂核心

- 宋钦宗赵桓：玩家。
- 李纲：主战守城核心。
- 种师道：西军老将。
- 宗泽：勤王与后续中兴核心。
- 主和宰执 2-3 人：推动和议、割地、罢李纲。
- 户部尚书：财政。
- 工部官：城防。
- 开封府尹：京城治理。
- 皇城司使：密令。
- 台谏代表 2 人：弹劾与清议。
- 宗室代表 1-2 人：南迁和质子压力。

### 军事与地方

- 殿前司将领 2 人。
- 禁军基层代表 1 人。
- 西军将领 2 人。
- 河东守臣 2 人。
- 河北边臣 2 人。
- 勤王地方官 2 人。

### 民间与财税

- 太学生代表。
- 京城商户代表。
- 转运使 2 人。
- 粮仓官 1 人。

人物设计要求：

- 每人至少一个立场。
- 每人至少一个秘密或弱点。
- 每人至少一个可改变命运的节点。

## 开发里程碑

### Milestone 1：题材换骨

目标：在当前架构上跑通一局最小靖康循环。

内容：

- 新内容文件。
- 新提示词。
- 基础人物、地区、军队、事件。
- 国库/内帑/京城粮/民心/君威。
- 召见、拟旨、结算可用。

验收：

- 玩家能新开局。
- 能召见李纲/户部/皇城司。
- 能下守城或查账旨意。
- 月末有回奏和数值变化。

### Milestone 2：爽点闭环

目标：做出“查证 → 对质 → 清算 → 反馈”的完整体验。

内容：

- 密令查账。
- 证据系统。
- 殿前对质界面。
- 裁断后财政和派系变化。
- 人物记忆。

验收：

- 禁军欠饷案能从事件发展到对质。
- 玩家能出示证据并裁断。
- 裁断影响国库、禁军、派系和后续事件。

### Milestone 3：汴京围城

目标：让军事压力成为第二条核心循环。

内容：

- 汴京守备。
- 城门节点。
- 金军威压。
- 夜攻战报。
- 勤王响应。
- 金宋谈判。

验收：

- 金军攻城结果受前期准备影响。
- 玩家能通过修城、补饷、派将、议和改变战局。
- 失败和胜利都有清晰原因。

### Milestone 4：UI 定型

目标：形成《朕不南渡》自己的视觉身份。

内容：

- 北宋舆图主界面。
- 宋代奏札风 UI。
- 朝臣席。
- 对质专属动效。
- 月末回奏版式。

验收：

- 首屏能立即看出题材和危机。
- 玩家能快速找到下一步操作。
- 爽点界面有仪式感。

### Milestone 5：内容扩写与平衡

目标：一局可重复游玩。

内容：

- 100+ 事件。
- 30+ 人物。
- 多个结局。
- 历史锚点分支。
- 数值平衡。
- 自动跑局测试。
- 路线覆盖报告。
- 早崩与数值飞天检测。

验收：

- 至少 5 条不同通关/失败路径。
- 玩家每局关键事件顺序不完全相同。
- 不存在单一万能策略。
- 标准自动测试中，前 5 回合失败率和路线胜率处于健康区间。
- 新手局失败时能输出可复盘原因和下一局建议。

## 风险与对策

### 风险：历史太沉重，爽点不足

对策：

- 每条危机都设计可清算对象或可逆转节点。
- 战报明确承认玩家准备的作用。
- 失败也产出线索。

### 风险：LLM 幻觉历史或乱改数值

对策：

- 历史人物和事件走内容文件。
- 结构化 extractor schema 严格限制字段。
- 钱粮必须走账本。
- 关键人物状态变化需要事件或旨意来源。

### 风险：系统太复杂，新玩家不知道干什么

对策：

- 首屏只显示 3-5 条急奏。
- 每条急奏提供“召见谁/可查什么/可拟什么”。
- 前三回合做隐性引导，不用长教程。

### 风险：清算太强，玩家一路杀穿

对策：

- 证据不足不能重判。
- 重判有派系反扑。
- 恐惧过高导致怠工、毁证、抱团。
- 杀错人会永久损害主战派或民心。

### 风险：主和派被脸谱化

对策：

- 设计合理主和动机：保城、保民、拖时间。
- 让某些主和建议在特定局势下确实有用。
- 让主战冒进也可能导致惨败。

## 立项建议

第一版不要追求“完整北宋”。只做一个足够锋利的短局：

```text
金兵来了，汴京快破了。
你先问谁，先查谁，先保哪道门，先杀还是先忍？
```

只要这个问题好玩，《朕不南渡》就站住了。
