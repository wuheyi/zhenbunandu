# 强派系系统设计

派系系统要把《朕不南渡》的官僚机器做活。派系不是阵营标签，也不是简单好感度，而是围绕职位、钱粮、名望、军权、地域、姻亲、门生、旧恩和恐惧形成的利益网络。

核心目标：

- 玩家每条路线都要触动某些人，也能借力某些人。
- 圣旨执行要经过派系控制的机构和人物，执行偏差有来源。
- 清算能带来爽点和短期服从，但会留下仇怨、毁证、怠工和抱团。
- 派系会根据玩家过去的承诺、惩罚、失信和胜败改变策略。
- 派系不是固定敌人。主和派可用来拖延，主战派也可能逼玩家冒进。

## 设计原则

1. **利益先于立场**：派系口号可以是忠义、保民、守城、和议，底层驱动力是利益和风险。
2. **人物挂在网络上**：一个人物可以有主派系、次级关系、门生债、人情债和恐惧来源。
3. **反扑有阶段**：不满不会瞬间爆炸，而是从议论、弹劾、拖延、泄密走向公开对抗。
4. **恐惧不是忠诚**：恐惧能压住短期执行，过高会引发逃亡、毁证、抱团和暗中卖国。
5. **胜利会改写站队**：守城胜利、金军退兵、追饷成功会让观望者转向；战败会让主和压力暴涨。

## 派系不是一维好感

每个派系至少有以下数值：

| 数值 | 范围 | 含义 |
| --- | --- | --- |
| influence | 0-100 | 能影响多少官员、机构、地方、舆论或军队 |
| affinity | -100 到 100 | 对皇帝当前路线的亲疏 |
| grievance | 0-100 | 积怨，来自清算、失信、利益受损 |
| fear | 0-100 | 恐惧，来自重判、抄没、军事胜利、皇威 |
| cohesion | 0-100 | 内部凝聚力，越高越能抱团行动 |
| exposure | 0-100 | 被查到把柄的程度，越高越容易被压制 |
| opportunity | 0-100 | 当前是否有机会反扑，如战败、粮价、君威低 |
| resources | 0-100 | 可调动的钱、人、线人、舆论、军权 |

建议 UI 不裸露全部数值，只显示：

- 亲疏。
- 反弹。
- 恐惧。
- 把柄。
- 当前动作。

## 派系构成

### 首批派系

| 派系 | 核心利益 | 控制资源 | 红线 | 可利用点 |
| --- | --- | --- | --- | --- |
| 主战清议 | 名节、守城、拒割地 | 台谏、太学生、舆论、李纲声望 | 罢忠臣、割地、屈辱和议 | 提升君威和战意，压主和派 |
| 主和宰执 | 保宗庙、避决战、保职位 | 中书门下、部分宰执、内廷沟通 | 让其背锅、公开羞辱、冒险野战 | 拖延金军，缓冲财政和民心 |
| 旧党门生 | 保旧账、保门生、保财路 | 财税官僚、地方人情、账册 | 查旧账、抄没、追赃 | 短期行政经验，知道钱在哪 |
| 转运财税网络 | 保路线、保油水、保商户信用 | 转运司、仓场、粮商、漕运 | 封账、强征、低价夺粮 | 筹粮筹钱能力强 |
| 宦官军功集团 | 保军权、保旧功、保宫中通道 | 禁军旧关系、内廷、军器仓 | 剥夺军权、查童贯旧账 | 熟悉禁军和军器，能快办脏活 |
| 西军边臣 | 要饷、要信任、要节制权 | 西军、边防声望、老将网络 | 空头许诺、文臣掣肘、拖欠军饷 | 真正能打，能救河东和汴京 |
| 宗室内廷 | 保血脉、保体面、保财富 | 宫中、宗室、内帑压力 | 让宗室为质、动内库太狠 | 可提供内帑、合法性背书 |
| 京城士民 | 粮价、治安、守城信心 | 太学生、坊市、民壮、舆论 | 粮价失控、滥抓、弃城 | 提供民壮、战意和清议 |
| 商户粮行 | 信用、价格、运输安全 | 粮食、借款、市场消息 | 强买、赖账、滥抄 | 快速借款和调粮 |
| 河北河东地方 | 保地方、保军民、求援 | 地方军、堡寨、路网情报 | 朝廷放弃、强征不救 | 牵制金军、提供屏障 |

### 派系关系

派系之间要有边：

| 关系 | 说明 |
| --- | --- |
| alliance | 临时或长期同盟 |
| rivalry | 争权或理念冲突 |
| patronage | 门生故吏关系 |
| debt | 人情债或财政债 |
| blackmail | 把柄关系 |
| channel | 互通消息的渠道 |

示例：

```text
主和宰执 ↔ 宗室内廷：共同害怕城破，容易推动南迁。
主战清议 ↔ 京城士民：守城口号一致，但粮价高时会分裂。
旧党门生 ↔ 转运财税网络：账册和财路互相保护。
西军边臣 ↔ 主战清议：路线相近，但西军要实际军饷，不吃空话。
```

## 人物的派系归属

人物不只属于一个派系。

字段建议：

| 字段 | 说明 |
| --- | --- |
| primary_faction | 主派系 |
| secondary_factions | 次级关系 |
| obligations | 人情债、举荐恩、姻亲、旧部 |
| fear_sources | 怕谁、怕什么证据 |
| ambition | 想要什么职位或结果 |
| red_lines | 绝不愿做什么 |
| leverage_on | 他握有谁的把柄 |
| leveraged_by | 谁握有他的把柄 |

示例：

```text
某户部官：
primary_faction = 转运财税网络
secondary_factions = 旧党门生
obligations = 曾受某宰执举荐
fear_sources = 军饷账册缺口
red_lines = 公开封账、牵连家族
```

这样玩家罢免他，不只是少一个官，而是触动一张网。

## 派系反应计算

建议使用可解释的评分，而不是黑箱随机。

```text
interest_damage = 诏令触犯本派核心利益的强度
interest_gain = 诏令满足本派诉求的强度
legitimacy = 证据强度 + 君威 + 战时必要性
shock = 恐惧带来的短期压制
opportunity = 战败、粮价、君威低、金军逼城等机会

reaction_pressure =
    interest_damage * 1.8
  + grievance * 0.8
  + opportunity * 0.7
  + cohesion * 0.4
  - interest_gain * 1.2
  - legitimacy * 0.9
  - min(fear, 65) * 0.5
  + max(0, fear - 75) * 0.7
```

解释：

- 恐惧在 65 以内压制反扑。
- 恐惧超过 75 反而推动极端自救。
- 证据和君威能让清算更稳。
- 战败或粮价暴涨会给反对派机会。

## 反扑阶段

每个派系可有多个反扑时钟。时钟不是一满就必然坏事，而是触发事件。

| 阶段 | 反应 | 典型事件 |
| --- | --- | --- |
| 0 | 观望 | 私下议论、称病 |
| 1 | 试探 | 上札反对、拖延文书 |
| 2 | 阻滞 | 拨款打折、粮运迟滞、执行变形 |
| 3 | 串联 | 多人弹劾、泄露密令、毁证 |
| 4 | 公开对抗 | 逼宫、请罢重臣、拒绝承办 |
| 5 | 危机事件 | 南迁风波、禁军哗变、粮行罢市、城门内应 |

### 反扑时钟示例

| 时钟 | 牵涉派系 | 推进来源 | 爆发结果 |
| --- | --- | --- | --- |
| 罢李纲风波 | 主和宰执、宗室内廷 | 任李纲过权、守城扰民、战败 | 李纲被迫请辞或朝议危机 |
| 转运怠工 | 转运财税网络、旧党门生 | 查账、抄没、低价征粮 | 粮船迟滞、军饷缩水 |
| 南迁暗议 | 宗室内廷、主和宰执 | 金军威压、粮价、城防低 | 宗室逼宫，君威下降 |
| 太学生请愿 | 主战清议、京城士民 | 割地、罢忠臣、粮价 | 舆论爆发，主和派受压 |
| 西军怨望 | 西军边臣 | 空头许诺、欠饷、文臣掣肘 | 勤王迟缓或拒绝入城 |
| 粮行罢市 | 商户粮行、转运网络 | 强买、滥抄、赖账 | 粮价暴涨，民心下降 |

## 派系行动库

派系每回合根据压力和机会选择行动。

### 温和行动

- 私下求见。
- 递札劝谏。
- 推荐折中方案。
- 要求补偿。
- 要求替罪羊。

### 阻滞行动

- 文书迟发。
- 预算打折。
- 承办人称病。
- 地方回称道路不通。
- 账册缺页。

### 攻击行动

- 弹劾关键人物。
- 放出流言。
- 泄露密令。
- 抢先毁证。
- 鼓动太学生或宗室。
- 勾连金使扩大压力。

### 极端行动

- 逼宫请和。
- 私开城门传闻。
- 禁军哗变。
- 宗室出逃。
- 粮商闭市。
- 边军拒诏。

## 玩家制衡手段

### 安抚

- 给职位。
- 给预算。
- 给面子。
- 采纳部分建议。
- 公开表彰。

适合降低 `grievance`，但可能提高对方 `influence`。

### 分化

- 提拔派系内部次要人物。
- 公开区别首恶与胁从。
- 私下许诺部分利益。
- 让两个派系互相查账。

适合降低 `cohesion`。

### 威慑

- 殿前对质。
- 下狱。
- 抄没。
- 斩首。
- 展示证据。

适合提高 `fear` 和 `exposure`，但会增加 `grievance`。

### 借力

- 借主战清议压主和派。
- 借皇城司查旧党账。
- 借商户粮行绕开转运。
- 借西军声望稳守城路线。

适合短期突破，但会欠人情。

### 兑现承诺

- 补发军饷。
- 按期给赏。
- 不滥杀胁从。
- 守住前线后兑现任命。

适合长期提高 `affinity` 和降低未来成本。

## 派系参与执行链

每道诏令都要经过若干执行节点：

```text
皇帝 → 中书/枢密 → 主管衙门 → 承办人 → 地方/军队/仓场 → 回奏
```

每个节点可能由不同派系控制。

示例：

```text
补发禁军欠饷：
皇帝
  → 户部：主和宰执影响
  → 转运司：财税网络控制
  → 殿前司：宦官军功集团旧关系
  → 禁军营：欠饷士卒
```

如果玩家只写“补饷”，但没有指定：

- 钱从哪里来。
- 谁押送。
- 谁核验。
- 谁监督。

则派系有更多空间截留或拖延。

## 派系与证据

证据不只打击人物，也打击派系。

证据效果：

- 提高涉事派系 `exposure`。
- 降低该派系反扑正当性。
- 让中立派不敢公开支持它。
- 允许玩家做更重裁断。

证据不足的清算：

- `fear` 上升。
- `grievance` 大幅上升。
- 主战清议或士民也可能不满。
- 后续“罗织忠良”事件概率上升。

## 派系记忆

派系要记住：

- 玩家清算过谁。
- 玩家救过谁。
- 玩家是否兑现承诺。
- 玩家是否战胜金军。
- 玩家是否割地或拒使。
- 玩家是否把某派当替罪羊。

记忆示例：

```json
{
  "subject_type": "faction",
  "subject_id": "转运财税网络",
  "event_type": "punishment",
  "title": "东仓账册案后多人被抄没",
  "sentiment": "negative",
  "importance": 5,
  "tags": ["查账", "抄没", "军饷", "旧党"]
}
```

这些记忆会影响：

- 召见时的口风。
- 朝议时的站队。
- 诏令执行折扣。
- 反扑时钟推进。
- 结局史论。

## UI 设计

### 派系总览

显示为“朝局图”：

- 中央是皇帝路线。
- 周围是派系节点。
- 节点大小代表影响力。
- 边颜色代表亲疏。
- 外圈火候代表反弹。
- 小印标记当前动作：拖、弹、密、乱、援。

### 派系详情

字段：

- 当前态度。
- 核心诉求。
- 红线。
- 控制资源。
- 关键人物。
- 当前反扑时钟。
- 最近记忆。
- 可用制衡手段。

### 诏令预览

拟旨时展示派系影响：

```text
查封转运司账册
支持：主战清议 +8，京城士民 +4
受损：转运财税网络 -18，旧党门生 -12
风险：粮运迟滞，中
建议：同时发布安抚商户榜文，指定皇城司护账，户部另派核验
```

| `faction_memberships` | 人物与派系关系，多对多 |

| `faction_edges` | 派系间关系 |

| `faction_clocks` | 反扑时钟 |
| `faction_reactions` | 每回合反应记录 |
| `faction_promises` | 玩家对派系承诺 |
| `faction_logs` | 数值变化和原因 |

## MVP 范围

首版不必做满复杂网络，但至少要有：

1. 8 个核心派系。
2. 每派 5 个数值：影响力、亲疏、积怨、恐惧、反弹。
3. 每派 1-2 个反扑时钟。
4. 人物主派系和次级派系。
5. 拟旨时的派系影响预览。
6. 月末派系反应回奏。
7. 派系记忆。

先让玩家感到“我不是在和数值玩，我是在和一张会记仇的官僚网周旋”。
