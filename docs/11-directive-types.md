# 丰富诏令类型设计

诏令系统是《朕不南渡》的操作核心。玩家不是点按钮，而是在有限信息、有限资源、有限时间里写下能被官僚机器执行的命令。

设计目标：

- 让“圣旨不是按钮，执行才是战场”落到具体玩法。
- 让每种诏令都有适用场景、必填字段、资源成本、执行链路和反噬风险。
- 让玩家能写脑洞命令，但系统能把它拆成结构化行动。
- 让同一个目标可以用不同文书形式实现，产生不同政治效果。

## 诏令由两层构成

```text
文书形式 + 行动类型 = 可执行诏令
```

例：

```text
公开榜文 + 安抚京城士民
密札 + 查封转运账册
军令 + 宣化门换防
手诏 + 召种师道入援
御前处分 + 罢某转运使并抄没
国书 + 假议和拖延金军
```

同样是“查账”，公开诏令会制造震慑，也会惊动对手；密札更隐蔽，但执行面窄，容易被承办人私吞结果。

## 文书形式

| 形式 | 公开度 | 速度 | 权威 | 风险 | 适用 |
| --- | --- | --- | --- | --- | --- |
| 圣旨 | 高 | 中 | 高 | 牵动大，难回头 | 重大任免、财政、军政路线 |
| 手诏 | 中 | 快 | 高 | 易被质疑程序 | 亲授重臣、紧急调度 |
| 中旨 | 低-中 | 快 | 中 | 士大夫反感 | 绕开常规机构 |
| 军令 | 中 | 快 | 高 | 需要军队服从 | 调兵、守城、军纪 |
| 密札 | 低 | 快 | 中 | 泄密后反噬 | 密查、反间、私下承诺 |
| 榜文 | 高 | 快 | 中 | 兑现不了伤信用 | 安民、动员、告示 |
| 敕牒 | 中 | 慢 | 中 | 官僚流程长 | 常规任免、授权 |
| 札子批答 | 中 | 中 | 中 | 受奏章框架限制 | 对大臣方案批示 |
| 国书 | 高 | 慢 | 高 | 外交后果大 | 议和、拒使、拖延 |
| 御前处分 | 高 | 快 | 高 | 派系反弹强 | 对质后裁断 |

## 诏令通用字段

每道诏令都应被结构化为：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| title | 是 | 简短标题 |
| form | 是 | 文书形式 |
| domain | 是 | 财政/军事/司法/外交/民政/人事等 |
| intent | 是 | 玩家真实意图 |
| public_text | 否 | 对外公开说法 |
| target | 是 | 人、机构、地区、军队、城门、事项 |
| assignee | 是 | 主承办人 |
| co_assignees | 否 | 协办人 |
| authority | 是 | 依据：皇命、战时、证据、朝议等 |
| resources | 条件必填 | 钱、粮、军资、民夫、职权 |
| deadline | 是 | 本回合/三日/一月/围城前 |
| evidence_refs | 条件必填 | 惩罚、查封、清算需证据 |
| execution_route | 是 | 执行链路 |
| success_condition | 是 | 怎样算办成 |
| fallback | 否 | 受阻时退而求其次 |
| risk_flags | 是 | 派系、民心、泄密、军事、财政风险 |
| faction_impacts | 是 | 预计影响哪些派系 |
| report_required | 是 | 谁回奏、何时回奏 |

## 执行阶段

诏令状态机：

```text
draft → reviewed → sealed → dispatched → in_execution → reported → completed
                                      ↘ distorted → partial → failed
                                      ↘ leaked → backlash
```

每道诏令月末回奏要说明：

- 原旨是什么。
- 经过哪些机构。
- 哪一层拖延、打折、误解或变形。
- 实际花了多少钱粮。
- 谁得利，谁受损。
- 留下什么隐患。

## 诏令类型总览

| 大类 | 子类 |
| --- | --- |
| 人事授权 | 任命、罢免、兼权、节制、召还、戴罪立功 |
| 财政钱粮 | 拨款、开内帑、追赃、查账、借款、加派、减免、平粜 |
| 军事调度 | 调兵、换防、补饷、练兵、出战、固守、勤王、军纪 |
| 城防工程 | 修城、筑壕、备器、储火油、征民夫、排查城门 |
| 司法监察 | 密查、封账、搜捕、对质、审讯、赦免、连坐限制 |
| 外交谋略 | 议和、拒使、拖延、质子、赎买、反间、离间金营 |
| 民政舆论 | 开仓、榜文、戒严、夜禁、禁谣、安抚太学、迁民 |
| 制度改革 | 战时军府、转运审计、禁军整编、奖惩制度、情报网 |

## 人事授权类

### 任命诏

用途：

- 任李纲总督城防。
- 任某人为京畿粮运使。
- 召种师道入援。

必填：

- 官职或临时权责。
- 权限边界。
- 任期或任务期限。
- 与原机构关系。

风险：

- 权力过大会被弹劾专权。
- 权限不清会被官僚互相推诿。
- 任错人会让事项恶化。

### 罢免诏

用途：

- 罢误事官。
- 解除某将兵权。
- 替换怯战承办人。

必填：

- 罢免理由。
- 继任者。
- 交接期限。
- 是否追责。

风险：

- 罢而无人接手会造成真空。
- 证据不足会被说成排除异己。
- 涉及大派系会推进反扑时钟。

### 兼权/节制诏

用途：

- 让一个人临时统合多部门。
- 战时减少扯皮。

必填：

- 统辖范围。
- 可调资源。
- 被节制机构。
- 回奏频率。

风险：

- 效率高，反弹也高。
- 被节制机构可能暗中拖延。

### 戴罪立功诏

用途：

- 抓住有能力但有罪的人继续使用。

必填：

- 罪名。
- 立功任务。
- 期限。
- 监督人。
- 失败处罚。

风险：

- 清议不满。
- 当事人可能借任务毁证。
- 成功后尾大不掉。

## 财政钱粮类

### 拨款诏

用途：

- 补发禁军欠饷。
- 修城。
- 赏勤王军。

必填：

- 金额。
- 来源账户。
- 接收对象。
- 押送和核验人。

风险：

- 没有核验会被截留。
- 国库不足会造成其他支出断裂。

### 开内帑诏

用途：

- 战时救急。
- 绕开户部拖延。

必填：

- 金额。
- 用途。
- 是否公开。
- 后续补还方案。

风险：

- 宗室内廷不满。
- 公开后会抬高各方要价。

### 查账/封账诏

用途：

- 查军饷。
- 查粮运。
- 查赔款去向。

必填：

- 账册范围。
- 封存地点。
- 主查人。
- 保护证据方式。

风险：

- 惊动涉案派系。
- 财税流程短期停摆。

### 追赃/抄没诏

用途：

- 清算贪腐，筹措急用。

必填：

- 证据。
- 涉案范围。
- 抄没边界。
- 入库账户。
- 防止滥抄的监督人。

风险：

- 爽点强，反扑也强。
- 滥抄商户会伤粮运信用。

### 借款诏

用途：

- 向商户、寺观、宗室、地方借款。

必填：

- 借款对象。
- 抵押或还款承诺。
- 利息或赏格。
- 违约处理。

风险：

- 失信会长期损害商户和地方响应。

### 平粜/开仓诏

用途：

- 压粮价。
- 安抚京城士民。

必填：

- 开仓数量。
- 售价或赈济规则。
- 维持秩序者。
- 防止冒领和囤积的办法。

风险：

- 粮储下降。
- 执行混乱会变成抢粮。

## 军事调度类

### 调兵诏

用途：

- 调西军入援。
- 调禁军换防。
- 派兵护粮。

必填：

- 部队。
- 主将。
- 路线。
- 粮草。
- 到达期限。
- 遇敌原则。

风险：

- 无粮则迟缓。
- 路线危险会折损。
- 调走某地兵力会留下空档。

### 守城军令

用途：

- 指定城门守将。
- 设置预备队。
- 加强夜防。

必填：

- 城门/城段。
- 驻守兵力。
- 器械。
- 守将。
- 换防时间。

风险：

- 守将不可信会提高城门风险。
- 过度戒严伤民心。

### 出战/袭营军令

用途：

- 主动打击金军。
- 断敌粮道。

必填：

- 目标。
- 兵力。
- 主将。
- 撤退条件。
- 接应安排。

风险：

- 失败会大幅提高主和压力。
- 胜利能显著提升君威和谈判筹码。

### 勤王诏

用途：

- 号召地方军入援。

必填：

- 召集对象。
- 路线。
- 赏格。
- 粮草接应。
- 到达地点。

风险：

- 空头勤王会损害朝廷信用。
- 各路军可能观望或被金军截击。

### 军纪诏

用途：

- 禁军扰民。
- 民壮混乱。
- 城门守军怯战。

必填：

- 禁止行为。
- 奖惩规则。
- 执法人。
- 申诉或核验方式。

风险：

- 过严会激起哗变。
- 过宽会纪律崩坏。

## 城防工程类

### 修城诏

必填：

- 城段。
- 材料。
- 民夫或工匠来源。
- 工期。
- 保护施工安排。

风险：

- 征发过重伤民心。
- 工部缺料会拖延。

### 备器诏

用途：

- 炮石、床弩、火油、拒马、沙袋。

必填：

- 器械种类。
- 数量。
- 存放地点。
- 负责人。

风险：

- 火油提高火患。
- 军器仓可能虚报。

### 排查城门诏

用途：

- 查内应。
- 查夜值。

必填：

- 城门。
- 主查人。
- 查验范围。
- 防止扰民规则。

风险：

- 误抓会伤民心。
- 泄密会让内应逃走。

## 司法监察类

### 密查札

用途：

- 暗查某人、某账、某营。

必填：

- 怀疑点。
- 承办人。
- 保密等级。
- 截止日期。

风险：

- 泄密后证据被毁。
- 皇城司滥权会伤士大夫亲疏。

### 搜捕诏

用途：

- 抓逃官、内应、毁证者。

必填：

- 对象。
- 罪名或证据。
- 执行人。
- 是否可搜宅。

风险：

- 证据不足就是罗织。
- 搜捕牵涉宗室或大臣会引发政治危机。

### 殿前对质令

用途：

- 把证据转化为公开裁断。

必填：

- 案名。
- 被质询人。
- 证据。
- 旁听或参与人物。
- 可选裁断范围。

风险：

- 证据被反驳会伤君威。
- 对质中可能攀咬更大人物。

### 赦免/宽贷诏

用途：

- 安抚胁从。
- 分化派系。

必填：

- 赦免范围。
- 不赦首恶。
- 换取条件。

风险：

- 太宽会损害清议。
- 太窄不能分化。

## 外交谋略类

### 议和国书

用途：

- 拖延。
- 争取退兵。
- 换取城防时间。

必填：

- 条件底线。
- 使者。
- 可让步项。
- 不可让步项。
- 拖延目标。

风险：

- 主战清议不满。
- 金方识破后更强硬。

### 拒使诏

用途：

- 公开拒绝屈辱条件。

必填：

- 拒绝理由。
- 安民解释。
- 军事准备。

风险：

- 金军攻势上升。
- 若准备不足会变成鲁莽。

### 质子/人质诏

用途：

- 极端外交筹码。

必填：

- 人选。
- 交换条件。
- 安全承诺。
- 底线。

风险：

- 宗室内廷剧烈反弹。
- 君威可能下降。

### 反间密札

用途：

- 离间金营诸将。
- 散布假情报。

必填：

- 目标。
- 线人。
- 经费。
- 假信息。
- 验证方式。

风险：

- 失败会暴露密线。

## 民政舆论类

### 安民榜文

用途：

- 稳定粮价和民心。
- 宣布守城决心。

必填：

- 对百姓承诺。
- 配套行动。
- 兑现期限。

风险：

- 榜文空话会伤信用。

### 戒严/夜禁令

用途：

- 降低内应和混乱。

必填：

- 范围。
- 时段。
- 执行人。
- 例外规则。

风险：

- 民心下降。
- 商贸受损，粮价上升。

### 安抚太学令

用途：

- 管理清议。
- 借士论压主和或防止失控。

必填：

- 接见对象。
- 公开解释。
- 允许表达边界。

风险：

- 太学生被鼓动后可能难以收束。

## 制度改革类

### 战时军府诏

用途：

- 临时整合城防、军器、开封府、禁军。

必填：

- 统辖人。
- 权限。
- 期限。
- 监督人。

风险：

- 效率高，专权攻击也高。

### 转运审计诏

用途：

- 长期解决粮运和军饷黑洞。

必填：

- 审计范围。
- 核验人。
- 不停摆保障。
- 分期目标。

风险：

- 短期粮运下降。
- 财税网络反扑。

### 禁军整编诏

用途：

- 提升禁军长期战力。

必填：

- 整编范围。
- 淘汰标准。
- 补饷方案。
- 新训练负责人。

风险：

- 短期军心波动。
- 将领利益受损。

## 组合诏令

强行动不应只靠单道命令，而是组合。

### 禁军欠饷组合

```text
1. 密札：皇城司封存军饷账册。
2. 拨款诏：内帑先补五万贯，国库补十五万贯。
3. 查账诏：户部与开封府共同核验发放名册。
4. 军纪诏：补饷后严禁鼓噪扰民。
5. 安民榜文：说明军饷发放是为守城，不额外加派。
```

效果：

- 成功率高。
- 花费高。
- 牵涉派系多。
- 但能减少执行偏差。

### 宣化门守备组合

```text
1. 任命诏：李纲临督宣化门。
2. 备器诏：拨火油、炮石、床弩。
3. 修城诏：工部三日内补城。
4. 排查诏：开封府查夜值和内应。
5. 榜文：募民壮搬运，按日给粮。
```

### 假议和拖延组合

```text
1. 国书：遣使议和，争取十日。
2. 密札：催种师道入援。
3. 反间札：散布宋军援兵将至消息。
4. 修城诏：趁谈判加固城门。
5. 朝议说明：压住主战派不满，声明不割三镇。
```

## 拟旨校验规则

系统生成草案后应检查：

| 检查 | 失败提示 |
| --- | --- |
| 是否有承办人 | “无人承办，此旨出京即散。” |
| 是否有资源来源 | “所需钱粮未列来源。” |
| 惩罚是否有证据 | “证据不足，重判恐伤君威。” |
| 调兵是否有粮道 | “此军无粮草接应，恐迟缓。” |
| 是否触动派系红线 | “此旨将强烈触动转运财税网络。” |
| 期限是否合理 | “三日修成全段城墙不现实。” |
| 执行链是否闭合 | “户部拨款后无人押送核验。” |

校验不是禁止玩家，而是提醒风险。玩家可以强行盖印。

## UI 设计

拟旨界面分四层：

1. **自然语言意图**：玩家写“查禁军饷到底去哪了”。
2. **结构化草案**：系统拆成密查、封账、对质或查账。
3. **风险预览**：钱粮、派系、民心、军事风险。
4. **盖印确认**：玩家选择公开、密行、强制、折中。

### 草案卡片

每张卡显示：

```text
查封禁军军饷账册
形式：密札
承办：皇城司使
目标：户部军饷账、殿前司发放册
期限：三日
风险：泄密中，旧党反扑低
缺口：未指定账册封存地点
```

### 朱批强度

玩家可选择执行强度：

| 强度 | 效果 |
| --- | --- |
| 宽 | 反弹低，执行慢 |
| 常 | 平衡 |
| 急 | 执行快，出错和扰民增加 |
| 严 | 威慑强，反扑和冤错风险高 |

## 月末回奏模板

每道诏令回奏格式：

```text
【原旨】
命皇城司三日内封存禁军军饷账册。

【执行】
皇城司夜入东仓，得账册二匣；户部主事称病不出，转运判官试图转移副册。

【偏差】
原拟秘密封账，因开封府差役走漏消息，旧党门客已有警觉。

【结果】
取得强证据一件，禁军欠饷案可开殿前对质。
转运财税网络积怨上升，毁证时钟推进。
```

## MVP 范围

首版至少实现 16 个模板：

1. 任命诏。
2. 罢免诏。
3. 戴罪立功诏。
4. 拨款诏。
5. 开内帑诏。
6. 查账密札。
7. 追赃抄没诏。
8. 平粜开仓诏。
9. 调兵诏。
10. 守城军令。
11. 勤王诏。
12. 修城诏。
13. 排查城门诏。
14. 殿前对质令。
15. 议和国书。
16. 安民榜文。

这 16 个足够支撑“问政、查案、守城、筹粮、外交、清算”的核心循环。
