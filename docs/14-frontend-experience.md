# 前端体验设计

《朕不南渡》的前端体验目标是让玩家像在御案前处理一台正在失控的国家机器，而不是在后台表格里改数值。界面要服务四个核心动作：

- 看清天下哪里在坏。
- 读懂奏疏里谁在说真话。
- 判断人物和派系关系。
- 看懂月末到底发生了什么。

本文重点设计地图交互、奏疏阅读、人物关系和月末报告。

## 总体交互原则

1. **先给结论，再给细节**：任何面板第一眼都要告诉玩家“急不急、影响谁、能做什么”。
2. **从信息到行动不超过两步**：看到危机后，能直接召见、密令、拟旨或加入本月重点。
3. **所有数字都能追溯**：资源变化、派系变化、城防变化都能点回来源事件或旨意。
4. **不让玩家迷路**：地图、奏疏、人物、报告之间始终保留上下文返回。
5. **古风不牺牲效率**：装饰服务气氛，信息密度和可读性优先。

## 主界面信息流

主界面不是四个孤立区域，而是一条连续工作流：

```text
朝报发现问题
  → 舆图定位问题
  → 召见相关人物
  → 读证据和奏疏
  → 形成草案
  → 颁诏
  → 月末报告复盘
  → 新问题回到朝报
```

任何卡片都应提供上下文动作：

- `定位舆图`
- `召见相关人`
- `立密令`
- `加入拟旨`
- `查看旧事`
- `标为本月重点`

## 地图交互

地图是玩家理解局势的中心，不是装饰背景。

### 地图模式

地图至少有 5 个模式：

| 模式 | 用途 |
| --- | --- |
| 北方局势 | 看金军、河东、河北、勤王路线 |
| 汴京城防 | 看城门、城墙、粮仓、皇城、金营 |
| 粮运财税 | 看粮道、漕运、仓场、税粮到达情况 |
| 派系势力 | 看各派系控制的机构、地区和人物 |
| 事件热区 | 看本月奏报、密令、战报集中在哪里 |

地图切换不能清空玩家上下文。比如玩家从“禁军欠饷”奏疏点进地图，应自动打开“粮运财税”层，并高亮相关路线和机构。

### 地图层级

每个地图模式由可开关图层组成。

#### 北方局势图层

- 金军路线。
- 宋军/勤王军路线。
- 关键城池。
- 战线压力。
- 预计到达时间。

#### 汴京城防图层

- 城门状态。
- 守军部署。
- 城防工程。
- 火患。
- 城门内应风险。
- 粮仓和水源。

#### 粮运财税图层

- 粮道。
- 漕运节点。
- 税粮来源。
- 仓场。
- 被截断或迟滞节点。
- 商户粮行影响区。

#### 派系势力图层

- 派系控制机构。
- 派系影响人物。
- 当前反扑时钟。
- 证据牵涉范围。

### 地图节点卡

点击地图节点，右侧或浮层打开节点卡。

节点卡结构：

```text
节点名：宣化门
一句话状态：城门稍稳，但火油不足，夜袭风险仍在。

关键指标：
城防 67 / 守军战意 58 / 内应风险 31 / 火患 44

相关：
守将：李纲
驻军：殿前司三千，民壮八百
相关事项：保卫汴京、排查城门内应
相关奏疏：金军夜探城壕

可行动：
拨火油 / 增派守军 / 排查夜值 / 召见李纲 / 查看城防日志
```

节点卡必须让玩家能立刻行动，不要只显示资料。

### 地图路线交互

路线不是静态线条。

点击路线显示：

- 起点和终点。
- 当前状态。
- 到达回合。
- 风险来源。
- 消耗。
- 负责人物。
- 最近阻碍。

示例：

```text
江淮粮道 → 汴京
预计到达：2 回合
运输量：12 万石
风险：盗匪 18 / 金军截击 31 / 转运截留 46
承办：京畿转运使
可行动：派兵护粮 / 查账 / 提高运费 / 改道
```

### 时间预览

地图要支持“下回合预览”：

- 金军箭线推进一格。
- 勤王军显示预计位置。
- 粮船显示预计到达或延误。
- 事项危机显示可能爆发点。

预览不需要精确模拟，只要让玩家知道“如果不管，会往哪里坏”。

### 地图与诏令联动

拟旨时，地图要同步高亮执行链：

```text
开内帑补禁军
内帑 → 户部 → 殿前司 → 禁军营
```

如果链路中某节点被高风险派系控制，地图上标“疑”或“阻”。

### 地图验收标准

- 玩家 5 秒内能看出金军主压力方向。
- 点击任何危机节点，都能找到至少一个可行动按钮。
- 地图路线变化能和月末报告中的结果互相跳转。
- 围城阶段能清楚区分“哪个城门危险，为什么危险”。

## 奏疏阅读体验

奏疏是玩家理解官僚机器的入口。它不能只是长文本，也不能被简化成任务列表。

### 奏疏列表

左栏默认展示本月 3-5 条急奏。

每条卡片显示：

```text
类型：边报 / 财报 / 台谏 / 密奏 / 战报
标题：禁军欠饷再起鼓噪
急迫：本月
可信：中
牵涉：户部、殿前司、转运财税网络
建议：召见户部 / 密查账册 / 补发急饷
```

卡片状态：

- 未读。
- 已读。
- 已标重点。
- 已转密令。
- 已转草案。
- 已结案。

### 奏疏详情

奏疏详情分四层：

1. **白话摘要**：一句话告诉玩家发生了什么。
2. **原文风格**：宋代奏疏口吻，保留沉浸感。
3. **结构化解读**：牵涉人物、派系、地区、资源、风险。
4. **玩家动作**：召见、追问、密令、拟旨、存入证据。

示例结构：

```text
摘要
禁军称三月未得全饷，夜间已有鼓噪。

奏疏原文
臣闻殿前诸军昨夜聚语...

解读
可信度：中
矛盾点：户部称已拨，殿前司称未足额收到
牵涉：户部、转运司、殿前司
风险：禁军士气下降，城防执行打折

可行动
召见户部 / 召见殿前司 / 密查账册 / 拟补饷诏
```

### 标注和摘录

玩家可以把奏疏中的句子标为：

- 线索。
- 证词。
- 风险。
- 承诺。
- 待核。

标注后进入“证据/线索夹”，可在召见和对质中引用。

这会让奏疏阅读从“看文本”变成“办案”。

### 可信度表达

可信度不要直接写成绝对真假。

使用：

- 已核。
- 多源相符。
- 单方奏报。
- 与账册矛盾。
- 牵涉自身利益。
- 疑有反间。

这样玩家会自然想去查证。

### 奏疏与历史记忆

奏疏详情中显示相关旧事：

```text
相关旧事：
上月户部承诺补发禁军急饷，但实际只拨三成。
两月前转运司因粮船迟滞被申饬。
```

点击旧事可打开史册定位。

### 阅读验收标准

- 玩家不读古文原文也能理解要点。
- 喜欢沉浸的玩家可以读完整奏疏。
- 每条奏疏都能导向至少一个行动或一个疑问。
- 玩家能把奏疏内容转为证据、追问或草案。

## 人物关系体验

人物关系是玩家判断“信谁、用谁、查谁”的工具。

### 人物关系图

人物关系图不是社交网络玩具，而是政治判断面板。

节点类型：

- 人物。
- 派系。
- 机构。
- 军队。
- 地区。
- 案件。
- 证据。

边类型：

| 边 | 含义 |
| --- | --- |
| 统属 | 官职上下级 |
| 门生 | 举荐、师承、人情 |
| 同派 | 派系关系 |
| 政敌 | 弹劾或路线冲突 |
| 把柄 | 证据或秘密 |
| 承诺 | 玩家或人物承诺 |
| 怨恨 | 被罢、被罚、被背叛 |
| 协作 | 共同承办事项 |

### 人物卡

人物卡第一屏要显示判断信息：

```text
李纲
兵部侍郎 / 主战清议

一句话：敢任事，能守城，但正被主和派攻击。

可用：
城防统筹、朝议主战、安抚士民

风险：
授权过大将触发“专权”弹劾

最近记忆：
上月奉诏修宣化门，工部拨料不足仍推进三成。
```

详细页再显示属性、关系、旧事和可用工具。

### 信任不是好感

人物页面不要只显示好感条。建议显示：

- 对皇帝路线的支持。
- 对当前任务的适配。
- 是否可信。
- 是否有把柄。
- 是否有派系牵绊。
- 是否敢担责。

例：

```text
可信：中高
担责：高
派系牵绊：主战清议
当前任务适配：守城高，财税低
```

### 关系筛选

玩家可按问题筛选关系图：

- 谁影响李纲被罢？
- 禁军欠饷案牵涉谁？
- 哪些人能压主和派？
- 哪些机构控制粮运？
- 谁握有转运司账册？

筛选结果要能一键：

- 召见。
- 立密令。
- 加入对质。
- 查看相关奏疏。

### 人物关系与对话联动

召见时，右侧显示“可引用关系”：

```text
你可追问：
- 他与转运判官有门生关系。
- 他上月曾承诺三日内补饷。
- 皇城司密报称其门客出入东仓。
```

玩家点击即可插入追问。

### 人物关系验收标准

- 玩家能从人物卡判断“这个人适合办什么事”。
- 玩家能追溯某个失败事项背后的人际和派系阻力。
- 关系图能导向行动，而不是只供观赏。
- 人物旧事能影响召见语气和执行结果。

## 月末报告体验

月末报告是玩家感到“我的决策改变历史”的关键。它要比普通日志更有仪式感，也更可复盘。

### 报告结构

月末报告分 7 段：

1. **本月总评**：一句话概括局势。
2. **大事时间线**：按顺序列出关键结果。
3. **诏令执行**：逐道旨意展示原意、执行、偏差、结果。
4. **战线与地图变化**：金军、勤王、粮道、城门变化。
5. **财政与民心**：钱粮流水和社会反应。
6. **人物与派系**：谁得势、谁记仇、谁开始反扑。
7. **下月预警**：最需要处理的 3 件事。

### 首屏总评

报告打开第一屏只显示：

```text
靖康元年二月回奏
本月京城稍稳，禁军鼓噪暂息；然转运账册案已惊动旧党，金军前锋逼近黄河。

大势：
汴京守备 +8
金军威压 +11
京城粮 -4 万石
主和压力 +6
```

下方提供按钮：

- 看执行详情。
- 看地图变化。
- 看派系反应。
- 进入下月。

### 诏令执行卡

每道诏令一张执行卡：

```text
补发禁军急饷

原旨：
以内帑五万贯、国库十五万贯补发殿前司欠饷。

执行：
内帑如数拨出，户部以库空迟拨五万，开封府核验营册时发现缺额。

偏差：
转运司扣留路费二万贯，称旧欠未清。

结果：
禁军欠饷下降 18，士气上升 9。
获得线索：东仓副册。
```

每张卡要有：

- `定位来源`
- `查看相关人`
- `追查偏差`
- `转为下月事项`

### 地图变化回放

月末报告中嵌入小型地图回放：

```text
金军路线推进
河东告急
江淮粮船迟滞
宣化门城防提升
```

玩家可以点任一变化，跳回地图节点详情。

### 数值变化追溯

每个数值变化都可以展开：

```text
君威 +4
  +6 殿前处分转运判官，朝堂震动
  +2 李纲守城见效
  -4 户部迟拨暴露圣旨执行不畅
```

这样玩家不会觉得数值是系统随手改的。

### 派系反应报告

派系报告要写清：

- 谁支持。
- 谁受损。
- 谁在观望。
- 谁开始反扑。
- 为什么。

示例：

```text
转运财税网络：积怨上升，恐惧上升。
原因：东仓账册被封，转运判官下狱。
后续：粮船迟滞风险上升，毁证时钟推进。

主战清议：亲疏上升。
原因：李纲守城获用，禁军补饷见效。
后续：若下月议和割地，将强烈反弹。
```

### 下月预警

报告最后不要丢一堆新事件，要给“下月三急”：

```text
下月三急：
1. 金军逼近黄河，需决定是否增援河东。
2. 东仓账册案可开殿前对质，但转运网络已有警觉。
3. 京城粮价继续上涨，若不开仓或调粮，民心将跌破 50。
```

每条可直接：

- 标重点。
- 召见相关人。
- 立密令。
- 生成草案。

### 报告验收标准

- 玩家能在 30 秒内知道本月赚了什么、亏了什么、下月该管什么。
- 每个失败结果都有原因，而不是“执行失败”四个字。
- 玩家能从报告直接进入下一步行动。
- 报告能形成“我这一局的史书”，可截图、可复盘、可传播。

## 跨界面联动

前端最重要的不是每个界面漂亮，而是界面之间能互相追溯。

### 标准跳转

| 来源 | 可跳转到 |
| --- | --- |
| 奏疏 | 地图节点、人物、密令、草案 |
| 地图节点 | 奏疏、人物、事项、诏令 |
| 人物卡 | 关系图、旧事、召见、证据 |
| 月末报告 | 诏令、地图变化、派系、史册 |
| 证据 | 奏疏原文、对质、涉案人物 |
| 事项 | 历史推进、承办人、相关诏令 |

### 面包屑

所有详情页保留来源：

```text
月末报告 > 补发禁军急饷 > 转运司扣留路费 > 东仓副册
```

玩家可以一步步返回，不丢上下文。

## 移动与小屏策略

如果屏幕较窄，主界面改成底部标签：

- 朝报。
- 舆图。
- 朝臣。
- 御案。
- 史册。

地图详情、奏疏详情、人物卡、月末报告都用全屏面板。

小屏必须保留：

- 顶栏核心资源。
- 当前最大危机。
- 底部主要操作。

## 前端状态与保存

前端应保存这些 UI 状态：

- 当前地图模式和图层。
- 本月重点标记。
- 奏疏已读状态。
- 玩家标注的线索。
- 关系图筛选。
- 月末报告阅读位置。

这些状态不一定影响游戏结算，但影响体验连续性。

## 实现优先级

### P0

- 主界面三栏工作流。
- 地图节点卡。
- 奏疏列表和详情。
- 人物卡。
- 月末报告基础结构。
- 跨界面跳转。

### P1

- 地图路线交互。
- 奏疏标注。
- 人物关系图。
- 数值变化追溯。
- 派系反应报告。

### P2

- 地图时间预览。
- 月末地图回放。
- 关系图高级筛选。
- 报告截图/分享版。
- 史册式长线复盘。

## 总体验收

前端体验合格时，玩家会形成清晰节奏：

```text
看奏疏发现问题，
点地图确认位置，
召见人物查清关系，
拟旨指定人钱期限，
月末报告看执行偏差，
再从偏差进入下一轮调查。
```

这就是《朕不南渡》的前端核心手感。
