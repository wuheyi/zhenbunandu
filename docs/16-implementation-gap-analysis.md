# 实现对照与下一步功能清单

本文按 `docs/09-mvp-roadmap.md`、`docs/13-new-player-onboarding.md`、`docs/14-frontend-experience.md` 对照当前代码实现，记录已经落地的能力、体验缺口和本轮实施范围。

## 当前已实现

### P0 核心循环

- 新局、本地 SQLite 存档、LLM 配置与无 key 规则结算。
- 朝报、召见、草案、拟旨、颁诏、月末流式结算。
- 国库、内帑、京城粮、民心、君威等核心资源。
- 汴京守备、金军威压、主和压力、勤王响应、粮价指数。
- 事项进度条、密令、证据、殿前对质、结局判定。
- 宣化门夜攻战报，且受守城、补饷、勤王等前置准备影响。

### P1 雏形

- 朝议多人辩论，并能生成勤王、议和、平粜三道草案。
- 财政流水表已入库，补饷、月末结算、追赃会写账。
- 城门节点、粮运路线、金宋谈判、勤王路线已有数据与地图展示。
- 史册回放与人物记忆已有基础记录。
- 固定脚本自动跑局、坏局种子扰动、早崩检测和平衡报告已有命令行雏形。
- 粮运与勤王路线已有节点级状态、节点处置和商户信用雏形。

## 主要缺口

### 新手引导与复盘

设计文档要求首局渐进提示、隐性保护和失败后可复盘原因。此前实现只有可玩流程，没有把“下一步为什么这么做”结构化呈现出来。

本轮已补：

- `/api/game/state` 返回 `guidance`，包含阶段、当前优先事项、可跳转行动和风险旗标。
- `/api/game/state` 返回 `postmortem`，结局后列出关键原因和下一局建议。
- 前端新增“御前小注”面板和回奏中的“失败复盘”区块。

### 人物关系判断

设计文档要求玩家能判断“信谁、用谁、查谁”。此前只有朝臣列表和召见，没有关系与派系温度的集中视图。

本轮已补：

- 新增“人物档案”面板。
- 展示人物基础能力、同组关系、证据牵涉、相关派系反扑和亲疏。
- 继续使用占位头像，不生成真实人物图片。

### 财政与战区追溯

设计文档要求钱粮流水、地区/军队详情、月末报告能追溯变化来源。此前数据已存在，但缺少完整读法。

本轮已补：

- 新增“财政流水”面板，展示国库、内帑、京城粮与流水明细。
- 新增“地区与军队详情”面板，覆盖地区、军队、城门、路线四类卡片。
- 月末回奏新增“变化追溯”，关联指标、粮运路线和派系反扑。

### 派系反扑与路线行动

设计文档要求派系不只是静态好感/反扑数值，而要能形成可见压力；粮运和勤王路线也应可被玩家直接处置。此前路线详情主要是读数，派系反扑也未进入回合链条。

本轮已补：

- 新增 `faction_clocks` 表与种子数据，覆盖罢李纲、转运怠工、南迁暗议、西军怨望、粮行罢市。
- 月末结算会根据诏令、缺口和时钟阶段触发怠工、观望、粮价、主和压力等反噬。
- 新增路线行动 API：护送、查账、保价、改道、勤王加赏、遣使催促。
- 新增反扑时钟缓和 API：人物谱中可安抚或揭示反扑链条。
- 前端“人物档案”展示反扑时钟，“地区与军队详情”的路线页提供可点选行动。

### 金宋谈判树

设计文档要求外交不是“和平按钮”，而是与军事、钱粮、主和压力和勤王互相牵动。此前只有 `diplomacy_state` 读数和“遣使假议和”诏令，玩家不能直接比较不同谈判策略。

本轮已补：

- `/api/game/state` 返回 `diplomacy_options`，按当前条件、国库、内帑、筹码和金方诉求生成可执行策略。
- 新增 `/api/diplomacy/action`，支持假议和拖延、犒军纳赂、强硬拒使、分化金营、拒割地质子。
- 外交行动会即时改动金军威压、主和压力、守军战意、谈判筹码、金营内讧、钱粮流水和派系时钟。
- 前端新增“金宋谈判”面板，展示金方当前要求、谈判数值和每项策略的收益、代价、风险。
- 继续使用现有占位地图和 UI 资源，没有生成真实外交或人物图片。

### 自动跑局与平衡报告

设计文档要求每天能用固定代理检查“靖康机器”有没有早崩、无脑爽游或路线失衡。此前测试只覆盖手写切片，无法比较多路线和固定坏局种子。

本轮已补：

- 新增 `zhenbunandu.autoplay`，支持守城、清算、议和拖延、勤王、民生、铁腕、平衡、随机 8 类固定脚本自动玩家。
- 按 `docs/12-autoplay-balancing.md` 的 8 个回归种子写入初始压力扰动，覆盖财政亏空、粮价危机、主和高压、派系反扑、金军急攻、勤王迟缓和城门风险。
- 每局使用临时 SQLite 存档运行，不污染玩家本地进度。
- 每回合记录核心指标、路线风险、派系时钟、外交状态、行动列表、回奏时间线和异常标记。
- `python -m zhenbunandu.autoplay --output runs/balance/latest` 会生成 `summary.md`、`metrics.csv`、`endings.json`、`route_compare.md`、`anomaly_report.md` 和 `sample_turns/`。
- 测试已覆盖坏局种子开局压力和批量报告产物。

### 勤王与粮运节点

设计文档要求路线不是抽象收入，而是可被截击、堵塞、贪污、拖延和接应的链条。此前已有路线整体风险、ETA 和行动，但缺少节点级读法，玩家看不清“到底卡在哪一段”。

本轮已补：

- 新增 `route_nodes`，为江淮粮道、京畿短途征粮和陕西西军勤王路配置仓场、漕运、粮行、渡口、河东接应、汴京接援等节点。
- `/api/game/state` 的每条 `logistics_routes` 会返回 `nodes`、`merchant_credit`、`blocked_nodes`，保持原路线字段兼容。
- `/api/routes/{route_id}/action` 支持可选 `node_id`，新增固护节点、清截击、接粮/接援、恢复信用等节点级行动。
- 月末派系反扑与诏令结算会推进或恶化对应节点，转运怠工、西军怨望和粮行罢市不再只改整体数值。
- 前端“地区与军队详情 > 路线”展示节点状态、节点风险、进度和节点动作按钮；继续使用现有占位地图，不生成真实路线图片。
- 测试已覆盖节点种子、勤王清截击、粮行信用恢复和 API 节点动作。

## 仍需后续实现

### 内容规模

- 人物仍只有 MVP 小样，距离文档中的约 30 人不足。
- 事件库只有首批开局与历史锚点，距离 100+ 事件不足。
- 结局路线仍偏少，尚未覆盖南迁、反攻、长期中兴等分支。

### 系统深度

- 派系反扑已有可见时钟、怠工和缓和入口，但毁证、抱团、互相牵连和长期记忆报复仍较浅。
- 金宋谈判已有可选策略和即时反馈，但仍缺少多回合条款履约、撤军/再犯、扣押使节、金营人物和正式条约记忆。
- 勤王路线已有节点、截击和接应雏形，但尚未模拟多路援军、不同主将性格、入城战役和援军入城后的政治代价。
- 粮运路线已有节点和商户信用雏形，但商户信用还没有长期记忆、地下囤积、票据体系和水路改造项目。

### 前端体验

- 地图仍是占位 SVG 与固定路线，没有真实地图资产和时间预览动画。
- 人物关系图是结构化列表，尚未做成可筛选的关系图谱。
- 月末地图回放和报告截图/分享版未实现。

### 自动平衡

- 已有固定脚本自动跑局和本地报告，但尚未接入 CI 定时跑、前端人工复盘面板、报告趋势对比和阈值失败门禁。
- 当前种子是规则扰动，不是完整剧本分歧；后续还需要随机种子池、真实 LLM 自动玩家和更长回合内容覆盖。

## 建议下一批实施顺序

1. 扩展事件与人物到 2-3 回合的稳定内容量。
2. 将自动跑局接入 CI 或本地固定脚本，建立前 5 回合早崩阈值。
3. 深化派系反扑：毁证、抱团、长期记忆报复和连锁议题。
4. 深化金宋谈判：多回合条款、撤军/再犯、扣使与金营人物。
5. 扩展勤王/粮运节点第二层：多路援军、入城战役、地下囤积、票据和水路改造。
6. 做自动跑局复盘面板：指标折线、失败前三回合、行动理由和代表局。
7. 最后再替换真实 UI 图片资产；在此之前继续使用占位图。
